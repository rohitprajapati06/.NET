# Error Pages Based on Status Code in ASP.NET Core MVC

In ASP.NET Core MVC, you can display different error pages based on the HTTP status code (404, 403, 500, etc.) instead of showing generic browser error messages.

This improves user experience and provides meaningful information when errors occur.

---

# Why Use Custom Error Pages?

Default browser error pages:

```text
404 Not Found
403 Forbidden
500 Internal Server Error
```

Custom error pages:

```text
404 - Page Not Found
The page you requested does not exist.

403 - Access Denied
You do not have permission to access this resource.

500 - Internal Server Error
Something went wrong on our side.
```

Benefits:

* Better user experience
* Professional appearance
* Branding consistency
* Easier troubleshooting
* Security (hides internal details)

---

# Handling Status Codes in ASP.NET Core MVC

ASP.NET Core provides:

```csharp
app.UseStatusCodePagesWithReExecute("/Error/{0}");
```

The `{0}` placeholder is replaced by the actual status code.

Examples:

```text
404 → /Error/404
403 → /Error/403
500 → /Error/500
```

---

# Configure in Program.cs

Place this before endpoint mapping:

```csharp
var app = builder.Build();

app.UseExceptionHandler("/Error/500");

app.UseStatusCodePagesWithReExecute("/Error/{0}");

app.UseStaticFiles();

app.UseRouting();

app.UseAuthentication();
app.UseAuthorization();

app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");

app.Run();
```

---

# Create Error Controller

```csharp
using Microsoft.AspNetCore.Mvc;

public class ErrorController : Controller
{
    [Route("Error/{statusCode}")]
    public IActionResult HandleError(int statusCode)
    {
        switch (statusCode)
        {
            case 404:
                return View("NotFound");

            case 403:
                return View("Forbidden");

            case 500:
                return View("ServerError");

            default:
                return View("Error");
        }
    }
}
```

# Handling Unhandled Exceptions

Status code pages do not handle exceptions.

Use:

```csharp
app.UseExceptionHandler("/Error/500");
```

When this exception occurs:

```csharp
public IActionResult Test()
{
    throw new Exception("Database Error");
}
```

ASP.NET Core redirects to:

```text
/Error/500
```

which loads:

```text
ServerError.cshtml
```

---

# Accessing Status Code Information

You can access the original request details.

```csharp
using Microsoft.AspNetCore.Diagnostics;

public IActionResult HandleError(int statusCode)
{
    var feature =
        HttpContext.Features.Get<IStatusCodeReExecuteFeature>();

    ViewBag.OriginalPath = feature?.OriginalPath;

    return statusCode switch
    {
        404 => View("NotFound"),
        403 => View("Forbidden"),
        _ => View("Error")
    };
}
```

# StatusCodePages Variations

## 1. Simple Text Response

```csharp
app.UseStatusCodePages();
```

Response:

```text
Status Code: 404
```

---

## 2. Redirect

```csharp
app.UseStatusCodePagesWithRedirects(
    "/Error/{0}");
```

Example:

```text
404 → Redirect → /Error/404
```

Browser URL changes.

---

## 3. Re-Execute (Recommended)

```csharp
app.UseStatusCodePagesWithReExecute(
    "/Error/{0}");
```

Example:

```text
404 → Re-execute → /Error/404
```

Browser URL remains unchanged.

---

# Redirect vs ReExecute

| Feature                | Redirect | ReExecute |
| ---------------------- | -------- | --------- |
| Browser URL Changes    | Yes      | No        |
| Extra HTTP Request     | Yes      | No        |
| Better User Experience | No       | Yes       |
| Recommended            | No       | Yes       |

---

# Example Flow

User requests:

```text
/products/9999
```

Product doesn't exist:

```csharp
return NotFound();
```

Pipeline:

```text
NotFound()
      ↓
404 Status Code
      ↓
UseStatusCodePagesWithReExecute
      ↓
/Error/404
      ↓
NotFound.cshtml
```

User sees:

```text

```

# Interview Questions

### What is `UseStatusCodePagesWithReExecute()`?

It intercepts non-success HTTP status codes and re-executes the request pipeline for a specified path without changing the browser URL.

---

### Difference between Redirect and ReExecute?

| Redirect              | ReExecute             |
| --------------------- | --------------------- |
| Creates a new request | Uses the same request |
| URL changes           | URL remains the same  |
| Less efficient        | More efficient        |

---

### Does `UseStatusCodePages()` handle exceptions?

No. It only handles HTTP status codes such as 404 and 403.

For exceptions, use:

```csharp
app.UseExceptionHandler("/Error/500");
```

---

### Which approach is recommended?

For MVC applications:

```text
UseExceptionHandler()
            +
UseStatusCodePagesWithReExecute()
            +
Custom Error Views
```

This combination handles both unhandled exceptions and HTTP status code errors in a clean, production-ready manner.
