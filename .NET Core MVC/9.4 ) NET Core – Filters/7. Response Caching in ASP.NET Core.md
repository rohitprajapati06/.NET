# Response Caching in ASP.NET Core

**Response Caching** is a technique used to improve application performance by storing the response of a request and serving the cached response for subsequent requests instead of executing the action again.

This reduces:

* Server processing time
* Database calls
* CPU usage
* Network traffic

and improves application response times.

---

# Why Use Response Caching?

Without caching:

```text
Request
   ↓
Controller
   ↓
Database Query
   ↓
Generate Response
   ↓
Send Response
```

For every request, the same processing occurs.

With caching:

```text
First Request
   ↓
Controller
   ↓
Database Query
   ↓
Store Response in Cache

Second Request
   ↓
Return Cached Response
```

No database call is required for the cached request.

---

# Response Caching Middleware

ASP.NET Core provides built-in Response Caching Middleware.

---

# Configure Response Caching

## Step 1: Register Service

### Program.cs

```csharp
builder.Services.AddResponseCaching();
```

---

## Step 2: Add Middleware

```csharp
app.UseResponseCaching();
```

Place it before endpoint mapping:

```csharp
app.UseRouting();

app.UseResponseCaching();

app.UseAuthentication();
app.UseAuthorization();

app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");
```

---

# Using ResponseCache Attribute

Apply the `ResponseCache` attribute to an action.

```csharp
using Microsoft.AspNetCore.Mvc;

public class HomeController : Controller
{
    [ResponseCache(Duration = 60)]
    public IActionResult Index()
    {
        return View();
    }
}
```

### Meaning

```csharp
[ResponseCache(Duration = 60)]
```

Cache the response for:

```text
60 Seconds
```

---

# Generated Headers

The attribute generates:

```http
Cache-Control: public,max-age=60
```

Browsers and proxies may cache the response.

---

# Example

```csharp
[ResponseCache(Duration = 30)]
public IActionResult Time()
{
    return Content(
        DateTime.Now.ToString());
}
```

### First Request

```text
10:00:01
```

### Second Request (within 30 seconds)

```text
10:00:01
```

Same cached response is returned.

---

# Cache by Query String

Suppose:

```text
/products?id=1
/products?id=2
```

Need different cache entries.

Use:

```csharp
[ResponseCache(
    Duration = 60,
    VaryByQueryKeys = new[] { "id" }
)]
public IActionResult Product(int id)
{
    return Content(
        $"Product {id}");
}
```

### Cache Entries

```text
/products?id=1
/products?id=2
```

Stored separately.

---

# Cache by Header

Example:

```csharp
[ResponseCache(
    Duration = 60,
    VaryByHeader = "User-Agent"
)]
public IActionResult Index()
{
    return View();
}
```

Different browsers receive separate cached responses.

---

# No Caching

Disable caching:

```csharp
[ResponseCache(
    NoStore = true,
    Location = ResponseCacheLocation.None)]
public IActionResult Privacy()
{
    return View();
}
```

Generated headers:

```http
Cache-Control: no-store,no-cache
Pragma: no-cache
```

Useful for:

* Login pages
* Payment pages
* Sensitive information

---

# ResponseCache Locations

## Any

```csharp
Location = ResponseCacheLocation.Any
```

Cached anywhere.

```http
Cache-Control: public
```

---

## Client

```csharp
Location = ResponseCacheLocation.Client
```

Cached only in the browser.

```http
Cache-Control: private
```

---

## None

```csharp
Location = ResponseCacheLocation.None
```

No caching allowed.

---

# Example with All Options

```csharp
[ResponseCache(
    Duration = 120,
    Location = ResponseCacheLocation.Client,
    VaryByHeader = "User-Agent")]
public IActionResult About()
{
    return View();
}
```

Meaning:

```text
Cache Duration: 120 seconds
Cache Location: Browser only
Different cache per User-Agent
```

# Response Caching vs In-Memory Caching

| Feature             | Response Caching     | In-Memory Caching |
| ------------------- | -------------------- | ----------------- |
| Caches              | Entire HTTP Response | Objects/Data      |
| Location            | Browser/Proxy        | Server Memory     |
| Requires Middleware | Yes                  | No                |
| Best For            | Static responses     | Database results  |
| Storage             | Client/Proxy         | Server            |

---

# In-Memory Cache Example

```csharp
private readonly IMemoryCache _cache;

public ProductService(IMemoryCache cache)
{
    _cache = cache;
}

public Product GetProduct(int id)
{
    return _cache.GetOrCreate(
        id,
        entry =>
        {
            entry.AbsoluteExpirationRelativeToNow =
                TimeSpan.FromMinutes(5);

            return GetProductFromDatabase(id);
        });
}
```

This caches data, not the HTTP response.

---

# Response Caching vs Output Caching

| Feature           | Response Caching | Output Caching |
| ----------------- | ---------------- | -------------- |
| Introduced        | ASP.NET Core 1   | ASP.NET Core 7 |
| Stores Data       | Client/Proxy     | Server         |
| Performance       | Good             | Better         |
| Recommended Today | Limited          | Yes            |
| Server-side Cache | No               | Yes            |

---

# Output Caching (Modern Approach)

Since ASP.NET Core 7+, Output Caching is preferred.

### Register

```csharp
builder.Services.AddOutputCache();
```

### Middleware

```csharp
app.UseOutputCache();
```

### Action

```csharp
[OutputCache(Duration = 60)]
public IActionResult Index()
{
    return View();
}
```

Benefits:

* Server-side caching
* Better performance
* More control
* Works independently of browser cache

# Interview Questions

### What is Response Caching?

Response Caching stores HTTP responses and reuses them for subsequent requests to improve performance.

---

### How do you enable Response Caching?

```csharp
builder.Services.AddResponseCaching();

app.UseResponseCaching();
```

---

### Which attribute is used for Response Caching?

```csharp
[ResponseCache]
```

Example:

```csharp
[ResponseCache(Duration = 60)]
```

---

### What is `VaryByQueryKeys`?

It creates separate cache entries based on query string values.

```csharp
[ResponseCache(
    Duration = 60,
    VaryByQueryKeys = new[] { "id" })]
```

---

### Difference between Response Caching and Output Caching?

| Response Caching         | Output Caching  |
| ------------------------ | --------------- |
| Client/Proxy based       | Server-side     |
| Older approach           | Modern approach |
| Depends on cache headers | Independent     |
| Less control             | More control    |

---

## Recommendation for Modern ASP.NET Core Applications

For applications like your **TourNest**, **SmartHealthcare**, and **La Caffeine** projects:

```text
Response Caching      → Basic browser/proxy caching
IMemoryCache          → Cache database results
Distributed Cache     → Multi-server deployments
Output Caching        → Preferred for ASP.NET Core 7+
```

If you're preparing for interviews, know both **Response Caching** and **Output Caching**, but for new production applications, **Output Caching** is generally the recommended caching mechanism.
