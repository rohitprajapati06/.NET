# Action Filters in ASP.NET Core MVC

**Action Filters** are filters that execute **before and after an action method** is executed. They are commonly used for logging, validation, auditing, performance monitoring, and modifying action inputs or outputs.

Action Filters are one of the most frequently used filter types in ASP.NET Core MVC.

---

# Why Use Action Filters?

Without Action Filters:

```csharp
public IActionResult Create(Product product)
{
    _logger.LogInformation("Action Started");

    // Business Logic

    _logger.LogInformation("Action Completed");

    return View();
}
```

You would need to repeat the same code in multiple actions.

With Action Filters:

```csharp
public class LoggingActionFilter : IActionFilter
{
    // Logging logic here
}
```

The logic is centralized and reusable.

---

# Position in MVC Pipeline

```text
Request
   │
   ▼
Authorization Filter
   │
   ▼
Resource Filter
   │
   ▼
Action Filter (Before)
   │
   ▼
Action Method
   │
   ▼
Action Filter (After)
   │
   ▼
Result Filter
   │
   ▼
Response
```

Action Filters execute immediately before and after the action method.

# Creating an Action Filter

Implement the `IActionFilter` interface.

```csharp
using Microsoft.AspNetCore.Mvc.Filters;

public class LoggingActionFilter : IActionFilter
{
    public void OnActionExecuting(
        ActionExecutingContext context)
    {
        Console.WriteLine("Before Action Execution");
    }

    public void OnActionExecuted(
        ActionExecutedContext context)
    {
        Console.WriteLine("After Action Execution");
    }
}
```

---

# IActionFilter Methods

## OnActionExecuting()

Runs before the action method executes.

```csharp
public void OnActionExecuting(
    ActionExecutingContext context)
{
}
```

Common uses:

* Validation
* Logging
* Authentication checks
* Auditing

---

## OnActionExecuted()

Runs after the action method executes.

```csharp
public void OnActionExecuted(
    ActionExecutedContext context)
{
}
```

Common uses:

* Logging
* Performance tracking
* Exception inspection

# Example: Logging Filter

```csharp
public class LoggingActionFilter : IActionFilter
{
    private readonly ILogger<LoggingActionFilter> _logger;

    public LoggingActionFilter(
        ILogger<LoggingActionFilter> logger)
    {
        _logger = logger;
    }

    public void OnActionExecuting(
        ActionExecutingContext context)
    {
        _logger.LogInformation(
            "Action Execution Started");
    }

    public void OnActionExecuted(
        ActionExecutedContext context)
    {
        _logger.LogInformation(
            "Action Execution Completed");
    }
}
```

---

# Example: Validation Filter

Instead of checking ModelState in every action:

```csharp
[HttpPost]
public IActionResult Create(Product product)
{
    if (!ModelState.IsValid)
    {
        return View(product);
    }

    return RedirectToAction("Index");
}
```

Use an Action Filter:

```csharp
public class ValidateModelFilter : IActionFilter
{
    public void OnActionExecuting(
        ActionExecutingContext context)
    {
        if (!context.ModelState.IsValid)
        {
            context.Result =
                new BadRequestObjectResult(
                    context.ModelState);
        }
    }

    public void OnActionExecuted(
        ActionExecutedContext context)
    {
    }
}
```

This automatically validates requests.

---

# Accessing Action Arguments

Action Filters can inspect parameters passed to actions.

```csharp
public void OnActionExecuting(
    ActionExecutingContext context)
{
    foreach (var argument in context.ActionArguments)
    {
        Console.WriteLine(
            $"{argument.Key}: {argument.Value}");
    }
}
```

Example:

```csharp
public IActionResult Details(int id)
{
    return View();
}
```

Output:

```text
id: 10
```

---

# Stopping Action Execution

You can short-circuit the pipeline.

```csharp
public void OnActionExecuting(
    ActionExecutingContext context)
{
    context.Result =
        new ContentResult
        {
            Content = "Access Denied"
        };
}
```

The action method will never execute.

# Async Action Filters

Implement `IAsyncActionFilter`.

```csharp
using Microsoft.AspNetCore.Mvc.Filters;

public class AsyncLoggingFilter
    : IAsyncActionFilter
{
    public async Task OnActionExecutionAsync(
        ActionExecutingContext context,
        ActionExecutionDelegate next)
    {
        Console.WriteLine("Before Action");

        await next();

        Console.WriteLine("After Action");
    }
}
```

---

# Understanding next()

```csharp
await next();
```

Represents the execution of:

```text
Action Method
      +
Remaining Filters
```

Flow:

```text
Before Action
      ↓
await next()
      ↓
Action Executes
      ↓
After Action
```

---

# Using ActionFilterAttribute

Instead of implementing `IActionFilter` directly:

```csharp
using Microsoft.AspNetCore.Mvc.Filters;

public class LoggingAttribute
    : ActionFilterAttribute
{
    public override void OnActionExecuting(
        ActionExecutingContext context)
    {
        Console.WriteLine("Before Action");
    }

    public override void OnActionExecuted(
        ActionExecutedContext context)
    {
        Console.WriteLine("After Action");
    }
}
```

Usage:

```csharp
[Logging]
public IActionResult Index()
{
    return View();
}
```

---

# Registering Action Filters

## Action Level

```csharp
[ServiceFilter(typeof(LoggingActionFilter))]
public IActionResult Index()
{
    return View();
}
```

---

## Controller Level

```csharp
[ServiceFilter(typeof(LoggingActionFilter))]
public class ProductController : Controller
{
}
```

Applies to all actions.

---

## Global Level

### Register Service

```csharp
builder.Services.AddScoped<LoggingActionFilter>();
```

### Add Globally

```csharp
builder.Services.AddControllersWithViews(options =>
{
    options.Filters.Add<LoggingActionFilter>();
});
```

Applies to every action in the application.

# Real-World Example: Performance Monitoring

```csharp
public class PerformanceFilter
    : IActionFilter
{
    private Stopwatch _stopwatch;

    public void OnActionExecuting(
        ActionExecutingContext context)
    {
        _stopwatch = Stopwatch.StartNew();
    }

    public void OnActionExecuted(
        ActionExecutedContext context)
    {
        _stopwatch.Stop();

        Console.WriteLine(
            $"Execution Time: " +
            $"{_stopwatch.ElapsedMilliseconds} ms");
    }
}
```

Useful for monitoring slow actions.

```csharp
    public void OnActionExecuted(
        ActionExecutedContext context)
    {
    }
}
```

# Action Filters vs Result Filters

| Feature                  | Action Filter | Result Filter |
| ------------------------ | ------------- | ------------- |
| Runs Before Action       | Yes           | No            |
| Runs After Action        | Yes           | No            |
| Runs Before Result       | No            | Yes           |
| Runs After Result        | No            | Yes           |
| Access Action Parameters | Yes           | No            |
| Modify Action Execution  | Yes           | No            |

---

# Action Filters vs Middleware

| Feature                   | Action Filter | Middleware |
| ------------------------- | ------------- | ---------- |
| MVC-Specific              | Yes           | No         |
| Access Controller Context | Yes           | No         |
| Access Action Arguments   | Yes           | No         |
| Entire Application        | No            | Yes        |

# Interview Questions

### What is an Action Filter?

An Action Filter executes before and after an action method and is used to implement reusable MVC-specific logic.

---

### Which interfaces are used to create Action Filters?

```csharp
IActionFilter
```

or

```csharp
IAsyncActionFilter
```

---

### What is the purpose of `OnActionExecuting()`?

It runs before the action method executes and is commonly used for validation, logging, and auditing.

---

### What is the purpose of `OnActionExecuted()`?

It runs after the action method executes and is commonly used for logging and performance monitoring.

---

### What does `context.Result` do in an Action Filter?

Setting `context.Result` short-circuits the pipeline and prevents the action method from executing.

Example:

```csharp
context.Result =
    new UnauthorizedResult();
```

---

### When should you use an Action Filter instead of Middleware?

Use an Action Filter when you need access to:

* Controller information
* Action parameters
* ModelState
* MVC-specific context

Use Middleware for application-wide request/response processing.

## Enterprise ASP.NET Core Usage

In applications such as **TourNest**, **SmartHealthcare**, and **La Caffeine**, Action Filters are commonly used for:

```text
✔ Audit Logging
✔ Request Validation
✔ Performance Monitoring
✔ User Activity Tracking
✔ Custom Business Rules
✔ Model Validation
```

A common production architecture is:

```text
Middleware
    ↓
Authorization Filters
    ↓
Action Filters
    ↓
Controllers
    ↓
Result Filters
    ↓
Response
```

where Action Filters handle reusable MVC-specific behavior close to controller execution.
