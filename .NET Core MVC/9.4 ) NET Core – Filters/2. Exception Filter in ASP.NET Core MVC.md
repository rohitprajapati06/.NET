# Exception Filter in ASP.NET Core MVC

An **Exception Filter** is a filter that executes when an unhandled exception occurs during the execution of a controller action or another MVC filter.

It provides a centralized way to handle exceptions and return a custom response instead of exposing application errors to users.

---

# Why Use Exception Filters?

Without exception filters:

```csharp
public IActionResult GetProduct(int id)
{
    var product = _productService.Get(id);

    if (product == null)
    {
        throw new Exception("Product not found");
    }

    return View(product);
}
```

When an exception occurs, the user may see a generic error page or the request may fail unexpectedly.

Using an Exception Filter:

* Log errors
* Return custom error messages
* Return custom JSON responses
* Prevent application crashes
* Centralize error handling

# Creating a Custom Exception Filter

Implement `IExceptionFilter`.

```csharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.Filters;

public class CustomExceptionFilter : IExceptionFilter
{
    public void OnException(ExceptionContext context)
    {
        context.Result = new JsonResult(new
        {
            Success = false,
            Message = "An unexpected error occurred."
        });

        context.ExceptionHandled = true;
    }
}
```

---

# Understanding the Code

```csharp
public void OnException(ExceptionContext context)
```

This method runs automatically whenever an unhandled exception occurs.

### Exception Information

```csharp
var exception = context.Exception;
```

Contains:

* Message
* Stack Trace
* Inner Exception
* Source

Example:

```csharp
string message = context.Exception.Message;
```

---

# ExceptionHandled Property

```csharp
context.ExceptionHandled = true;
```

Tells ASP.NET Core:

> "The exception has been handled. Do not rethrow it."

Without this:

```csharp
context.ExceptionHandled = false;
```

ASP.NET Core continues propagating the exception.

# Returning a View

Instead of JSON:

```csharp
public class CustomExceptionFilter : IExceptionFilter
{
    public void OnException(ExceptionContext context)
    {
        context.Result = new ViewResult
        {
            ViewName = "Error"
        };

        context.ExceptionHandled = true;
    }
}
```

### Error.cshtml

```html
<h2>Something went wrong.</h2>
<p>Please try again later.</p>
```

# Returning a Custom Status Code

```csharp
public class CustomExceptionFilter : IExceptionFilter
{
    public void OnException(ExceptionContext context)
    {
        context.Result = new ObjectResult(new
        {
            Message = "Server Error"
        })
        {
            StatusCode = 500
        };

        context.ExceptionHandled = true;
    }
}
```

Response:

```json
{
  "message": "Server Error"
}
```

HTTP Status:

```text

```

# Async Exception Filter

For asynchronous operations, implement `IAsyncExceptionFilter`.

```csharp
using Microsoft.AspNetCore.Mvc.Filters;

public class AsyncExceptionFilter : IAsyncExceptionFilter
{
    public async Task OnExceptionAsync(
        ExceptionContext context)
    {
        context.Result = new JsonResult(new
        {
            Message = "Error occurred"
        });

        context.ExceptionHandled = true;

        await Task.CompletedTask;
    }
}
```

---

# Registering Exception Filters

## 1. Action Level

```csharp
[ServiceFilter(typeof(CustomExceptionFilter))]
public IActionResult Index()
{
    throw new Exception("Test Error");
}
```

Only this action uses the filter.

---

## 2. Controller Level

```csharp
[ServiceFilter(typeof(CustomExceptionFilter))]
public class ProductController : Controller
{
}
```

Applies to all actions in the controller.

---

## 3. Global Level

### Register Service

```csharp
builder.Services.AddScoped<CustomExceptionFilter>();
```

### Add Filter Globally

```csharp
builder.Services.AddControllersWithViews(options =>
{
    options.Filters.Add<CustomExceptionFilter>();
});
```

Now every MVC action uses the filter.

# Logging Exceptions

A common use case is logging errors.

```csharp
public class CustomExceptionFilter : IExceptionFilter
{
    private readonly ILogger<CustomExceptionFilter> _logger;

    public CustomExceptionFilter(
        ILogger<CustomExceptionFilter> logger)
    {
        _logger = logger;
    }

    public void OnException(ExceptionContext context)
    {
        _logger.LogError(
            context.Exception,
            "Unhandled Exception Occurred");

        context.Result = new JsonResult(new
        {
            Message = "Something went wrong."
        });

        context.ExceptionHandled = true;
    }
}
```

# Example in ASP.NET Core Web API

```csharp
[HttpGet("{id}")]
public IActionResult GetProduct(int id)
{
    throw new Exception("Database error");
}
```

Exception Filter:

```csharp
public class ApiExceptionFilter : IExceptionFilter
{
    public void OnException(ExceptionContext context)
    {
        context.Result = new ObjectResult(new
        {
            Success = false,
            Error = context.Exception.Message
        })
        {
            StatusCode = 500
        };

        context.ExceptionHandled = true;
    }
}
```

Response:

```json
{
  "success": false,
  "error": "Database error"
}
```

# Exception Filter vs Middleware

| Feature                         | Exception Filter | Exception Middleware |
| ------------------------------- | ---------------- | -------------------- |
| Scope                           | MVC Only         | Entire Application   |
| Handles Controller Exceptions   | Yes              | Yes                  |
| Handles Middleware Exceptions   | No               | Yes                  |
| Access MVC Context              | Yes              | No                   |
| Recommended for Global Handling | No               | Yes                  |

---

# Exception Filter vs Global Exception Middleware

### Exception Filter

```text
MVC Only
Controller Exceptions Only
```

### Global Exception Middleware

```text
Whole Application
MVC + APIs + Middleware + Routing
```

Example:

```csharp
app.UseExceptionHandler("/Home/Error");
```

or a custom middleware:

```csharp
app.UseMiddleware<GlobalExceptionMiddleware>();
```

---

# Best Practice in Modern ASP.NET Core

For production applications:

✅ Use **Global Exception Middleware** for centralized error handling.

✅ Use **Exception Filters** only when you need MVC-specific exception handling.

Example:

```text
Global Exception Middleware
        +
Serilog Logging
        +
ProblemDetails Response
```

This is the preferred architecture in modern ASP.NET Core Web API and MVC applications.

# Interview Questions

### What is an Exception Filter?

An Exception Filter handles unhandled exceptions that occur during MVC action execution and allows custom error responses.

---

### Which interface is used to create an Exception Filter?

```csharp
IExceptionFilter
```

or

```csharp
IAsyncExceptionFilter
```

---

### What does `context.ExceptionHandled = true` do?

It tells ASP.NET Core that the exception has been handled and should not be rethrown.

---

### Can Exception Filters catch exceptions from Middleware?

No. Exception Filters only work within the MVC pipeline.

---

### Which is preferred for global exception handling?

**Global Exception Middleware** is preferred because it handles exceptions across the entire ASP.NET Core request pipeline, not just MVC actions.
