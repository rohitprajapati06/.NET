# Authorization Filters in ASP.NET Core MVC

**Authorization Filters** are the first filters that execute in the ASP.NET Core MVC filter pipeline. Their primary responsibility is to determine whether a user is authorized to access a controller or action.

If authorization fails, the request is short-circuited, and the action method is never executed.

---

# Why Use Authorization Filters?

Authorization filters help:

* Restrict access to authenticated users
* Implement role-based security
* Protect sensitive resources
* Prevent unauthorized access
* Centralize authorization logic

Example:

```text
Anonymous User
      ↓
Authorization Filter
      ↓
Access Denied (401/403)
```

The action method is never reached.

---

# Filter Execution Order

```text
Request
   │
   ▼
Authorization Filter
   │
   ▼
Resource Filter
   │
   ▼
Action Filter
   │
   ▼
Action Method
   │
   ▼
Result Filter
   │
   ▼
Response
```

Authorization filters execute before all other filters.

# Built-in Authorization Filters

ASP.NET Core provides built-in authorization attributes:

## Authorize Attribute

Restricts access to authenticated users.

```csharp
using Microsoft.AspNetCore.Authorization;

[Authorize]
public class ProductController : Controller
{
    public IActionResult Index()
    {
        return View();
    }
}
```

Only logged-in users can access the controller.

---

# Authorize Action Level

```csharp
public class ProductController : Controller
{
    [Authorize]
    public IActionResult Create()
    {
        return View();
    }

    public IActionResult Index()
    {
        return View();
    }
}
```

Only the `Create()` action requires authentication.

---

# Allow Anonymous

Overrides authorization requirements.

```csharp
[Authorize]
public class AccountController : Controller
{
    [AllowAnonymous]
    public IActionResult Login()
    {
        return View();
    }

    [AllowAnonymous]
    public IActionResult Register()
    {
        return View();
    }
}
```

Anyone can access Login and Register.

---

# Role-Based Authorization

Restrict access based on user roles.

```csharp
[Authorize(Roles = "Admin")]
public IActionResult Dashboard()
{
    return View();
}
```

Only users in the Admin role can access the action.

---

# Multiple Roles

```csharp
[Authorize(Roles = "Admin,Manager")]
public IActionResult Reports()
{
    return View();
}
```

Access is granted if the user belongs to:

```text
Admin OR Manager
```

# Multiple Authorize Attributes

```csharp
[Authorize(Roles = "Admin")]
[Authorize(Roles = "Manager")]
public IActionResult SecureArea()
{
    return View();
}
```

This means:

```text
Admin AND Manager
```

The user must belong to both roles.

# Policy-Based Authorization

### Program.cs

```csharp
builder.Services.AddAuthorization(options =>
{
    options.AddPolicy(
        "AdminOnly",
        policy => policy.RequireRole("Admin"));
});
```

### Controller

```csharp
[Authorize(Policy = "AdminOnly")]
public IActionResult AdminPanel()
{
    return View();
}
```

# Custom Authorization Filter

Implement `IAuthorizationFilter`.

```csharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.Filters;

public class CustomAuthorizationFilter
    : IAuthorizationFilter
{
    public void OnAuthorization(
        AuthorizationFilterContext context)
    {
        var user =
            context.HttpContext.User;

        if (!user.Identity.IsAuthenticated)
        {
            context.Result =
                new UnauthorizedResult();
        }
    }
}
```

---

# Understanding the Code

## Access Current User

```csharp
var user =
    context.HttpContext.User;
```

Gets the authenticated user.

---

## Check Authentication

```csharp
user.Identity.IsAuthenticated
```

Returns:

```text
true  → Logged In
false → Anonymous User
```

---

## Stop Request Execution

```csharp
context.Result =
    new UnauthorizedResult();
```

Returns:

```http
401 Unauthorized
```

The request pipeline stops.

---

# Custom Role Authorization Filter

```csharp
public class AdminFilter
    : IAuthorizationFilter
{
    public void OnAuthorization(
        AuthorizationFilterContext context)
    {
        if (!context.HttpContext.User
            .IsInRole("Admin"))
        {
            context.Result =
                new ForbidResult();
        }
    }
}
```

Response:

```http
403 Forbidden
```

---

# Async Authorization Filter

Implement `IAsyncAuthorizationFilter`.

```csharp
using Microsoft.AspNetCore.Mvc.Filters;

public class AsyncAuthorizationFilter
    : IAsyncAuthorizationFilter
{
    public async Task OnAuthorizationAsync(
        AuthorizationFilterContext context)
    {
        if (!context.HttpContext.User
            .Identity.IsAuthenticated)
        {
            context.Result =
                new UnauthorizedResult();
        }

        await Task.CompletedTask;
    }
}
```

# Registering Custom Authorization Filters

## Service Registration

```csharp
builder.Services.AddScoped<AdminFilter>();
```

---

## Apply to Action

```csharp
[ServiceFilter(typeof(AdminFilter))]
public IActionResult Dashboard()
{
    return View();
}
```

---

## Apply to Controller

```csharp
[ServiceFilter(typeof(AdminFilter))]
public class AdminController : Controller
{
}
```

---

## Apply Globally

```csharp
builder.Services.AddControllersWithViews(options =>
{
    options.Filters.Add<AdminFilter>();
});
```

Applies to all controllers.

# Custom Authorization Attribute

Create a reusable attribute.

```csharp
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.Filters;

public class AdminOnlyAttribute
    : Attribute,
      IAuthorizationFilter
{
    public void OnAuthorization(
        AuthorizationFilterContext context)
    {
        if (!context.HttpContext.User
            .IsInRole("Admin"))
        {
            context.Result =
                new ForbidResult();
        }
    }
}
```

Usage:

```csharp
[AdminOnly]
public IActionResult Settings()
{
    return View();
}
```

# Authorization Filter vs Action Filter

| Feature                  | Authorization Filter | Action Filter |
| ------------------------ | -------------------- | ------------- |
| Executes First           | Yes                  | No            |
| Controls Access          | Yes                  | No            |
| Can Stop Request         | Yes                  | Yes           |
| Used for Security        | Yes                  | No            |
| Access Action Parameters | No                   | Yes           |

---

# Authorization Filter vs Middleware

| Feature                      | Authorization Filter | Middleware |
| ---------------------------- | -------------------- | ---------- |
| MVC Specific                 | Yes                  | No         |
| Access MVC Context           | Yes                  | No         |
| Global Security              | Limited              | Yes        |
| Preferred for Authentication | No                   | Yes        |

---

# Authorization Filter vs Authorize Attribute

| Feature           | Authorize Attribute | Custom Authorization Filter |
| ----------------- | ------------------- | --------------------------- |
| Built-in          | Yes                 | No                          |
| Easy to Use       | Yes                 | No                          |
| Supports Roles    | Yes                 | Yes                         |
| Supports Policies | Yes                 | Custom Logic Needed         |
| Recommended       | Yes                 | Only for Special Cases      |

# Best Practices

### Prefer

✅ `[Authorize]`

✅ Role-based authorization

✅ Policy-based authorization

✅ Claims-based authorization

### Avoid

❌ Writing custom authorization filters unless necessary

❌ Hardcoding roles in multiple places

❌ Performing authentication inside controllers

# Interview Questions

### What is an Authorization Filter?

An Authorization Filter determines whether a user is allowed to access a controller or action before the action executes.

---

### Which interface is used to create a custom Authorization Filter?

```csharp
IAuthorizationFilter
```

or

```csharp
IAsyncAuthorizationFilter
```

---

### Which filter executes first in ASP.NET Core MVC?

**Authorization Filter** executes first in the filter pipeline.

---

### What is the difference between `UnauthorizedResult` and `ForbidResult`?

| Result                 | Status Code | Meaning                                    |
| ---------------------- | ----------- | ------------------------------------------ |
| `UnauthorizedResult()` | 401         | User is not authenticated                  |
| `ForbidResult()`       | 403         | User is authenticated but lacks permission |

---

### What is the recommended approach for authorization in ASP.NET Core?

Use:

```text
ASP.NET Core Identity
        +
JWT/Cookies Authentication
        +
[Authorize]
        +
Policies
        +
Roles/Claims
```

Custom Authorization Filters should only be used when you need specialized authorization logic that cannot be achieved using the built-in authorization system.
