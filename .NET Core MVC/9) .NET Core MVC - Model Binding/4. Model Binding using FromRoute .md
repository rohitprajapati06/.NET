# Model Binding using `[FromRoute]` in ASP.NET Core

`[FromRoute]` tells ASP.NET Core to bind values from the **URL route** to action method parameters or model properties.

For example:

```http
GET /api/doctors/123
```

Here:

```text
123
```

is part of the URL path (route), not the query string.

---

# Why Use `[FromRoute]`?

Without model binding, you would need to manually extract route values:

```csharp
var id = RouteData.Values["id"];
```

With `[FromRoute]`:

```csharp
[HttpGet("{id}")]
public IActionResult GetDoctor([FromRoute] int id)
{
    return Ok(id);
}
```

ASP.NET Core automatically assigns the route value to `id`.

---

# Example 1: Simple Route Parameter

### Controller

```csharp
[HttpGet("{id}")]
public IActionResult GetDoctor(
    [FromRoute] int id)
{
    return Ok(id);
}
```

### Request

```http
GET /api/doctors/123
```

### Result

```csharp
id = 123
```


### Result

```csharp
route.HospitalId = 1
route.DoctorId = 10
```

# Route Constraints

ASP.NET Core can validate route values before the action executes.

### Integer Constraint

```csharp
[HttpGet("{id:int}")]
public IActionResult GetDoctor(int id)
{
    return Ok(id);
}
```

Valid:

```http
GET /api/doctors/123
```

Invalid:

```http
GET /api/doctors/abc
```

Returns:

```http
404 Not Found
```

---

### GUID Constraint

```csharp
[HttpGet("{id:guid}")]
public IActionResult GetDoctor(Guid id)
{
    return Ok(id);
}
```

Valid:

```http
GET /api/doctors/8d5ef54f-11f6-4b7f-a2a0-7f66e3a0fd20
```

Invalid:

```http
GET /api/doctors/test
```

Returns:

```http
404 Not Found
```

# Real Example in SmartHealthcare

### Controller

```csharp
[ApiController]
[Route("api/hospitals")]
public class HospitalsController : ControllerBase
{
    [HttpGet("{hospitalId}")]
    public async Task<IActionResult> GetHospital(
        [FromRoute] Guid hospitalId)
    {
        var result = await mediator.Send(
            new GetHospitalByIdQuery(hospitalId));

        return Ok(result);
    }
}
```

### Request

```http
GET /api/hospitals/5c7b9b6d-3e4c-4a95-a7d3-c8d59df62a4f
```

### Result

```csharp
hospitalId =
5c7b9b6d-3e4c-4a95-a7d3-c8d59df62a4f
```

# `[FromRoute]` vs `[FromQuery]`

### FromRoute

```http
GET /api/doctors/123
```

```csharp
[HttpGet("{id}")]
public IActionResult GetDoctor(
    [FromRoute] int id)
{
}
```

Used when the value identifies a specific resource.

---

### FromQuery

```http
GET /api/doctors?id=123
```

```csharp
public IActionResult GetDoctor(
    [FromQuery] int id)
{
}
```

Used for filtering or searching.

---

# REST API Best Practice

### Get Single Resource

✅ Use Route

```http
GET /api/doctors/123
```

```csharp
[HttpGet("{id}")]
public IActionResult GetDoctor(
    [FromRoute] int id)
{
}
```

---

### Search Resources

✅ Use Query String

```http
GET /api/doctors?specialization=Cardiology
```

```csharp
public IActionResult SearchDoctors(
    [FromQuery] string specialization)
{
}
```

# Interview Answer

> `[FromRoute]` is a model binding attribute in ASP.NET Core that binds values from URL route parameters to action method parameters or model objects. It is commonly used to identify a specific resource in REST APIs, such as retrieving, updating, or deleting an entity by its ID.`
