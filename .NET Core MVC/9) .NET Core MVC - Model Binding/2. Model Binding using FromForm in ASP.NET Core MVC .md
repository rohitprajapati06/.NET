### Model Binding using `[FromForm]` in ASP.NET Core MVC

`[FromForm]` tells ASP.NET Core to bind data from **form fields** (`multipart/form-data` or `application/x-www-form-urlencoded`) to an action parameter or model.

It is commonly used when:

* Submitting HTML forms
* Uploading files (`IFormFile`)
* Sending form data from Postman
* AJAX requests using `FormData`

---

# Why `[FromForm]`?

Suppose a request contains:

```http
Content-Type: multipart/form-data
```

Form Data:

```text
FirstName = Rohit
LastName = Prajapati
Age = 24
```

ASP.NET Core can automatically map these values to a C# object.

---

## Example 1: Simple Form Binding

### DTO

```csharp
public class RegisterPatientDto
{
    public string FirstName { get; set; }

    public string LastName { get; set; }

    public int Age { get; set; }
}
```

### Controller

```csharp
[HttpPost]
public IActionResult Register(
    [FromForm] RegisterPatientDto dto)
{
    return Ok(dto);
}
```

### Form Data

```text
FirstName = Rohit
LastName = Prajapati
Age = 24
```

### Result

```csharp
dto.FirstName = "Rohit"
dto.LastName = "Prajapati"
dto.Age = 24
```

# Example 2: Uploading Files

This is the most common use case.

### DTO

```csharp
public class UploadCertificateDto
{
    public Guid DoctorId { get; set; }

    public IFormFile Certificate { get; set; }
}
```

### Controller

```csharp
[HttpPost]
public async Task<IActionResult> UploadCertificate(
    [FromForm] UploadCertificateDto dto)
{
    if (dto.Certificate != null)
    {
        var fileName = dto.Certificate.FileName;
        var length = dto.Certificate.Length;
    }

    return Ok();
}
```

### Postman

```text
DoctorId      = 123456
Certificate   = doctor.pdf
```

ASP.NET Core automatically binds:

```csharp
dto.DoctorId
dto.Certificate
```

# Example 3: Multiple Files

### DTO

```csharp
public class UploadDocumentsDto
{
    public Guid PatientId { get; set; }

    public List<IFormFile> Documents { get; set; }
}
```

### Controller

```csharp
[HttpPost]
public async Task<IActionResult> UploadDocuments(
    [FromForm] UploadDocumentsDto dto)
{
    foreach (var file in dto.Documents)
    {
        Console.WriteLine(file.FileName);
    }

    return Ok();
}
```

# Example 4: HTML Form

### View

```html
<form asp-action="Register" method="post">
    <input type="text" name="FirstName" />
    <input type="text" name="LastName" />
    <input type="number" name="Age" />

    <button type="submit">
        Submit
    </button>
</form>
```

### Controller

```csharp
[HttpPost]
public IActionResult Register(
    [FromForm] RegisterPatientDto dto)
{
    return View();
}
```

ASP.NET Core automatically maps the input names to DTO properties.

# Example 5: AJAX + FormData

```javascript
const formData = new FormData();

formData.append("FirstName", "Rohit");
formData.append("LastName", "Prajapati");
formData.append("Age", 24);

fetch("/api/patient/register", {
    method: "POST",
    body: formData
});
```

Controller:

```csharp
[HttpPost]
public IActionResult Register(
    [FromForm] RegisterPatientDto dto)
{
    return Ok(dto);
}
```

# How ASP.NET Core Matches Values

DTO:

```csharp
public class DoctorDto
{
    public string Name { get; set; }

    public string Specialization { get; set; }
}
```

Form Data:

```text
Name = Dr Rohit
Specialization = Cardiologist
```

ASP.NET Core matches:

| Form Field     | Property       |
| -------------- | -------------- |
| Name           | Name           |
| Specialization | Specialization |

and creates:

```csharp
new DoctorDto
{
    Name = "Dr Rohit",
    Specialization = "Cardiologist"
};
```

# `[FromForm]` vs `[FromBody]`

| Feature        | FromForm | FromBody |
| -------------- | -------- | -------- |
| HTML Forms     | ✅        | ❌        |
| File Upload    | ✅        | ❌        |
| JSON Requests  | ❌        | ✅        |
| Multipart Data | ✅        | ❌        |
| API Payloads   | Limited  | ✅        |

### `[FromBody]`

```json
{
  "name":"Rohit",
  "age":24
}
```

```csharp
public IActionResult Create(
    [FromBody] UserDto dto)
{
}
```

---

### `[FromForm]`

```text
name = Rohit
age = 24
```

```csharp
public IActionResult Create(
    [FromForm] UserDto dto)
{
}
```

# In Your SmartHealthcare Project

For uploading doctor certificates, hospital logos, medical reports, profile photos, etc., use:

```csharp
public class UploadDoctorDocumentCommand
{
    public Guid DoctorId { get; set; }

    public IFormFile Document { get; set; }
}
```

```csharp
[HttpPost("upload-document")]
public async Task<IActionResult> UploadDocument(
    [FromForm] UploadDoctorDocumentCommand command)
{
    return Ok();
}
```

This is the standard enterprise approach when handling files along with additional form fields.

### Interview Answer

> `[FromForm]` is a model binding attribute in ASP.NET Core that instructs the framework to bind action parameters or model properties from form data submitted in the HTTP request. It is primarily used for HTML form submissions and file uploads using `multipart/form-data` or `application/x-www-form-urlencoded` content types.
