# Custom Model Binding in ASP.NET Core

## What is Custom Model Binding?

Custom Model Binding allows you to define **how ASP.NET Core converts incoming request data into a .NET object** when the default model binding behavior is not sufficient.

Use Custom Model Binding when:

* Request format is unusual
* Data needs special parsing
* Values come from multiple sources
* You want automatic conversion to custom objects

---

# Why Do We Need Custom Model Binding?

Suppose your API receives:

```http
GET /api/doctors?doctor=1|Rohit|Cardiology
```

Default model binding cannot convert:

```text
1|Rohit|Cardiology
```

into:

```csharp
public class DoctorDto
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string Specialization { get; set; }
}
```

A Custom Model Binder can perform this conversion automatically.

---

# Step 1: Create Model

```csharp
public class DoctorDto
{
    public int Id { get; set; }

    public string Name { get; set; }

    public string Specialization { get; set; }
}
```

---

# Step 2: Create Custom Model Binder

A model binder implements `IModelBinder`.

```csharp
using Microsoft.AspNetCore.Mvc.ModelBinding;

public class DoctorModelBinder : IModelBinder
{
    public Task BindModelAsync(
        ModelBindingContext bindingContext)
    {
        var value = bindingContext
            .ValueProvider
            .GetValue(bindingContext.ModelName)
            .FirstValue;

        if (string.IsNullOrEmpty(value))
        {
            return Task.CompletedTask;
        }

        var parts = value.Split('|');

        var doctor = new DoctorDto
        {
            Id = int.Parse(parts[0]),
            Name = parts[1],
            Specialization = parts[2]
        };

        bindingContext.Result =
            ModelBindingResult.Success(doctor);

        return Task.CompletedTask;
    }
}
```

---

# Step 3: Use the Binder

```csharp
[HttpGet]
public IActionResult GetDoctor(
    [ModelBinder(BinderType = typeof(DoctorModelBinder))]
    DoctorDto doctor)
{
    return Ok(doctor);
}
```

# Real-World Example: Comma Separated GUIDs

A common enterprise scenario.

Request:

```http
GET /api/doctors?ids=guid1,guid2,guid3
```

Default model binding may not always match your desired format.

---

## Model

```csharp
public class GuidList
{
    public List<Guid> Ids { get; set; }
}
```

---

## Custom Binder

```csharp
public class GuidListBinder : IModelBinder
{
    public Task BindModelAsync(
        ModelBindingContext bindingContext)
    {
        var value = bindingContext
            .ValueProvider
            .GetValue("ids")
            .FirstValue;

        if (string.IsNullOrWhiteSpace(value))
        {
            return Task.CompletedTask;
        }

        var ids = value
            .Split(',')
            .Select(Guid.Parse)
            .ToList();

        bindingContext.Result =
            ModelBindingResult.Success(ids);

        return Task.CompletedTask;
    }
}
```

---

## Usage

```csharp
[HttpGet]
public IActionResult GetDoctors(
    [ModelBinder(BinderType = typeof(GuidListBinder))]
    List<Guid> ids)
{
    return Ok(ids);
}
```

# Creating a Binder Provider

Instead of adding `[ModelBinder]` everywhere, you can register a binder globally.

---

## Provider

```csharp
using Microsoft.AspNetCore.Mvc.ModelBinding;

public class DoctorModelBinderProvider
    : IModelBinderProvider
{
    public IModelBinder GetBinder(
        ModelBinderProviderContext context)
    {
        if (context.Metadata.ModelType ==
            typeof(DoctorDto))
        {
            return new DoctorModelBinder();
        }

        return null;
    }
}
```

---

## Register in Program.cs

```csharp
builder.Services.AddControllers(options =>
{
    options.ModelBinderProviders.Insert(
        0,
        new DoctorModelBinderProvider());
});
```

Now every `DoctorDto` automatically uses your custom binder.

# Binding from Multiple Sources

Suppose you want:

```http
GET /api/doctors/5?hospitalId=10
```

Route:

```text
doctorId = 5
```

Query:

```text
hospitalId = 10
```

Create:

```csharp
public class DoctorRequest
{
    public int DoctorId { get; set; }

    public int HospitalId { get; set; }
}
```

Custom binder:

```csharp
public class DoctorRequestBinder : IModelBinder
{
    public Task BindModelAsync(
        ModelBindingContext context)
    {
        var doctorId =
            context.HttpContext.Request.RouteValues["id"];

        var hospitalId =
            context.HttpContext.Request.Query["hospitalId"];

        var request = new DoctorRequest
        {
            DoctorId = Convert.ToInt32(doctorId),
            HospitalId = Convert.ToInt32(hospitalId)
        };

        context.Result =
            ModelBindingResult.Success(request);

        return Task.CompletedTask;
    }
}
```

This combines data from multiple request sources.

---

# Custom Binder vs Type Converter

For simple conversions, use `TypeConverter`.

Example:

```csharp
public class Money
{
    public decimal Amount { get; set; }
}
```

If only a string needs conversion:

```text
100.50
```

→

```csharp
new Money { Amount = 100.50m }
```

A `TypeConverter` is often simpler than a custom binder.

# Example for SmartHealthcare

Suppose every request contains:

```http
X-Hospital-Id: guid
```

Instead of reading headers everywhere:

```csharp
var hospitalId =
Request.Headers["X-Hospital-Id"];
```

Create:

```csharp
public class HospitalContext
{
    public Guid HospitalId { get; set; }
}
```

Custom binder:

```csharp
public class HospitalContextBinder
    : IModelBinder
{
    public Task BindModelAsync(
        ModelBindingContext context)
    {
        var hospitalId =
            context.HttpContext.Request.Headers["X-Hospital-Id"];

        var model = new HospitalContext
        {
            HospitalId = Guid.Parse(hospitalId)
        };

        context.Result =
            ModelBindingResult.Success(model);

        return Task.CompletedTask;
    }
}
```

Controller:

```csharp
[HttpGet]
public IActionResult GetDoctors(
    [ModelBinder(
        BinderType = typeof(HospitalContextBinder))]
    HospitalContext hospital)
{
    return Ok();
}
```

No manual header reading is needed.

# Interview Answer

> Custom Model Binding in ASP.NET Core is a mechanism that allows developers to define how incoming request data is converted into .NET objects when the default model binding behavior is insufficient. It is implemented by creating classes that implement `IModelBinder` and is commonly used for custom parsing, combining values from multiple request sources, strongly typed identifiers, encrypted values, and other specialized binding scenarios.
