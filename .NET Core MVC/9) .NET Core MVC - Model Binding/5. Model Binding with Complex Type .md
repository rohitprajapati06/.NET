# Model Binding with Complex Types in ASP.NET Core

A **Complex Type** is a class that contains one or more properties.

Instead of binding a single value like:

```csharp
[FromRoute] Guid id
```

ASP.NET Core can bind an entire object:

```csharp
public class RegisterDoctorDto
{
    public string Name { get; set; }
    public string Email { get; set; }
    public string Specialization { get; set; }
}
```

and automatically populate all properties from the incoming request.

---

# Simple Type vs Complex Type

### Simple Type

```csharp
public IActionResult GetDoctor(int id)
{
}
```

`int`, `string`, `Guid`, `bool`, `DateTime` are simple types.

---

### Complex Type

```csharp
public class DoctorDto
{
    public string Name { get; set; }
    public string Email { get; set; }
}
```

```csharp
public IActionResult CreateDoctor(DoctorDto dto)
{
}
```

`DoctorDto` is a complex type.

---

# Example 1: Complex Type From Request Body

### DTO

```csharp
public class RegisterDoctorDto
{
    public string Name { get; set; }

    public string Email { get; set; }

    public string Specialization { get; set; }
}
```

### Controller

```csharp
[HttpPost]
public IActionResult Register(
    [FromBody] RegisterDoctorDto dto)
{
    return Ok(dto);
}
```

### Request

```json
{
  "name": "Dr Rohit",
  "email": "rohit@test.com",
  "specialization": "Cardiology"
}
```

### Result

```csharp
dto.Name = "Dr Rohit";
dto.Email = "rohit@test.com";
dto.Specialization = "Cardiology";
```

ASP.NET Core creates the object and fills all properties.

# Example 2: Complex Type From Query String

### DTO

```csharp
public class DoctorSearchDto
{
    public string Name { get; set; }

    public string Specialization { get; set; }

    public int Page { get; set; }

    public int PageSize { get; set; }
}
```

### Controller

```csharp
[HttpGet]
public IActionResult SearchDoctors(
    [FromQuery] DoctorSearchDto dto)
{
    return Ok(dto);
}
```

### Request

```http
GET /api/doctors?
Name=Rohit&
Specialization=Cardiology&
Page=1&
PageSize=10
```

### Result

```csharp
dto.Name = "Rohit";
dto.Specialization = "Cardiology";
dto.Page = 1;
dto.PageSize = 10;
```

# Nested Complex Types

Complex types can contain other complex types.

### DTOs

```csharp
public class AddressDto
{
    public string City { get; set; }

    public string State { get; set; }
}

public class RegisterHospitalDto
{
    public string Name { get; set; }

    public AddressDto Address { get; set; }
}
```

### Request

```json
{
  "name": "City Hospital",
  "address": {
    "city": "Mumbai",
    "state": "Maharashtra"
  }
}
```

### Result

```csharp
dto.Name = "City Hospital";

dto.Address.City = "Mumbai";

dto.Address.State = "Maharashtra";
```

ASP.NET Core recursively binds nested objects.

# Collection Binding

### DTO

```csharp
public class DoctorDto
{
    public string Name { get; set; }
}

public class HospitalDto
{
    public string Name { get; set; }

    public List<DoctorDto> Doctors { get; set; }
}
```

### Request

```json
{
  "name": "Apollo",
  "doctors": [
    {
      "name": "Doctor A"
    },
    {
      "name": "Doctor B"
    }
  ]
}
```

### Result

```csharp
dto.Doctors.Count = 2;
```

ASP.NET Core automatically binds collections.

# Validation with Complex Types

### DTO

```csharp
public class RegisterDoctorDto
{
    [Required]
    public string Name { get; set; }

    [EmailAddress]
    public string Email { get; set; }
}
```

### Controller

```csharp
[HttpPost]
public IActionResult Register(
    RegisterDoctorDto dto)
{
    if (!ModelState.IsValid)
    {
        return BadRequest(ModelState);
    }

    return Ok(dto);
}
```

### Invalid Request

```json
{
  "name": "",
  "email": "invalid-email"
}
```

### ModelState

```json
{
  "Name": [
    "The Name field is required."
  ],
  "Email": [
    "The Email field is not a valid e-mail address."
  ]
}
```

# Complex Type Binding in `[ApiController]`

Most Web APIs use:

```csharp
[ApiController]
[Route("api/[controller]")]
public class DoctorsController : ControllerBase
{
}
```

With `[ApiController]`:

```csharp
[HttpPost]
public IActionResult CreateDoctor(
    RegisterDoctorDto dto)
{
}
```

You can omit `[FromBody]`.

ASP.NET Core automatically assumes:

```csharp
[FromBody] RegisterDoctorDto dto
```

for complex types.

# Real Example in SmartHealthcare

### Command

```csharp
public class RegisterDoctorCommand
{
    public string FirstName { get; set; }

    public string LastName { get; set; }

    public string Email { get; set; }

    public Guid HospitalId { get; set; }

    public string Specialization { get; set; }
}
```

### Controller

```csharp
[HttpPost]
public async Task<IActionResult> RegisterDoctor(
    RegisterDoctorCommand command)
{
    var result = await _mediator.Send(command);

    return Ok(result);
}
```

### Request

```json
{
  "firstName": "Rohit",
  "lastName": "Prajapati",
  "email": "rohit@test.com",
  "hospitalId": "6b1d2d7d-8c0b-4db0-90d8-bd7f95f7a123",
  "specialization": "Cardiology"
}
```

ASP.NET Core automatically builds the `RegisterDoctorCommand` object and passes it to MediatR.

# Interview Answer

> Complex Type Model Binding in ASP.NET Core is the process of automatically creating and populating a class object from incoming request data. The framework maps request values from the body, query string, route, or form fields to matching properties of the object, including nested objects and collections, reducing manual data extraction and object creation.
