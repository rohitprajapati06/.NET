# Model Binding using `[FromQuery]` in ASP.NET Core

`[FromQuery]` tells ASP.NET Core to bind data from the **query string** of the URL to action parameters or model objects.

A query string is the part of the URL after `?`.

Example:

```http
GET /api/doctors?specialization=Cardiology&page=1&pageSize=10
```

Here:

```text
specialization = Cardiology
page = 1
pageSize = 10
```

are query string parameters.

---

# Why Use `[FromQuery]`?

Instead of manually reading values:

```csharp
var specialization = Request.Query["specialization"];
```

ASP.NET Core automatically binds them.

```csharp
public IActionResult GetDoctors(
    [FromQuery] string specialization)
{
    return Ok(specialization);
}
```

---

# Example 1: Simple Query Parameter

### Controller

```csharp
[HttpGet]
public IActionResult Search(
    [FromQuery] string city)
{
    return Ok(city);
}
```

### Request

```http
GET /api/hospitals?city=Mumbai
```

### Result

```csharp
city = "Mumbai"
```

# Example 3: Query String to DTO

Instead of multiple parameters, use a DTO.

### DTO

```csharp
public class HospitalSearchDto
{
    public string City { get; set; }

    public string Specialization { get; set; }

    public int Page { get; set; }

    public int PageSize { get; set; }
}
```

### Controller

```csharp
[HttpGet]
public IActionResult Search(
    [FromQuery] HospitalSearchDto request)
{
    return Ok(request);
}
```

### Request

```http
GET /api/hospitals?
City=Mumbai&
Specialization=Cardiology&
Page=1&
PageSize=10
```

### Result

```csharp
request.City = "Mumbai"
request.Specialization = "Cardiology"
request.Page = 1
request.PageSize = 10
```

# Real Example in SmartHealthcare

### Hospital Search

```csharp
public class HospitalSearchQuery
{
    public string City { get; set; }

    public string State { get; set; }

    public string Specialization { get; set; }

    public int Page { get; set; } = 1;

    public int PageSize { get; set; } = 10;
}
```

### Controller

```csharp
[HttpGet("search")]
public async Task<IActionResult> SearchHospitals(
    [FromQuery] HospitalSearchQuery query)
{
    var result = await mediator.Send(query);

    return Ok(result);
}
```

### Request

```http
GET /api/hospitals/search?
City=Mumbai&
State=Maharashtra&
Specialization=Cardiology&
Page=1&
PageSize=10
```

ASP.NET Core automatically creates:

```csharp
new HospitalSearchQuery
{
    City = "Mumbai",
    State = "Maharashtra",
    Specialization = "Cardiology",
    Page = 1,
    PageSize = 10
};
```

# `[FromQuery]` vs `[FromRoute]`

### `[FromRoute]`

Route:

```http
GET /api/doctors/123
```

Controller:

```csharp
[HttpGet("{id}")]
public IActionResult GetDoctor(
    [FromRoute] Guid id)
{
}
```

Result:

```csharp
id = 123
```

---

### `[FromQuery]`

Route:

```http
GET /api/doctors?id=123
```

Controller:

```csharp
[HttpGet]
public IActionResult GetDoctor(
    [FromQuery] Guid id)
{
}
```

Result:

```csharp
id = 123
```

# When to Use `[FromQuery]`

Use `[FromQuery]` for:

✅ Searching data

```http
/api/doctors?name=Rohit
```

✅ Filtering data

```http
/api/doctors?specialization=Cardiology
```

✅ Pagination

```http
/api/doctors?page=1&pageSize=20
```

✅ Sorting

```http
/api/doctors?sortBy=Name
```

✅ Optional parameters

```http
/api/doctors?city=Mumbai
```

Avoid using it for large or complex objects; use `[FromBody]` for POST/PUT requests with JSON payloads.

---

# Interview Answer

> `[FromQuery]` is a model binding attribute in ASP.NET Core that binds values from the query string of an HTTP request URL to action method parameters or model objects. It is commonly used in GET APIs for searching, filtering, sorting, and pagination scenarios.
