# File Handling in ASP.NET Core MVC

File handling in ASP.NET Core MVC allows applications to **upload, download, store, retrieve, and delete files** such as images, PDFs, documents, videos, and other media.

Common use cases include:

* Profile picture uploads
* Resume uploads
* Product images
* Invoice generation
* Document management systems
* Azure Blob Storage integration

# File Upload in ASP.NET Core MVC

ASP.NET Core provides the **IFormFile** interface for handling uploaded files.

## Model

```csharp
public class UploadFileViewModel
{
    public IFormFile File { get; set; }
}
```

---

## View

```html
<form asp-action="Upload"
      method="post"
      enctype="multipart/form-data">

    <input type="file" name="File" />

    <button type="submit">
        Upload
    </button>

</form>
```

### Important

```html
enctype="multipart/form-data"
```

Without this attribute, files will not be uploaded.

---

## Controller

```csharp
public class FileController : Controller
{
    private readonly IWebHostEnvironment _environment;

    public FileController(IWebHostEnvironment environment)
    {
        _environment = environment;
    }

    [HttpPost]
    public async Task<IActionResult> Upload(
        UploadFileViewModel model)
    {
        if (model.File != null &&
            model.File.Length > 0)
        {
            string uploadsFolder =
                Path.Combine(
                    _environment.WebRootPath,
                    "uploads");

            Directory.CreateDirectory(
                uploadsFolder);

            string filePath =
                Path.Combine(
                    uploadsFolder,
                    model.File.FileName);

            using (var stream =
                new FileStream(
                    filePath,
                    FileMode.Create))
            {
                await model.File.CopyToAsync(stream);
            }

            ViewBag.Message =
                "File Uploaded Successfully";
        }

        return View();
    }
}
```

`IFormFile` represents a file sent with an HTTP request.

```csharp
model.File.FileName
```

```csharp

```

Returns file size in bytes.


# Uploading Multiple Files

## Model

```csharp
public class MultipleFileUploadViewModel
{
    public List<IFormFile> Files { get; set; }
}
```

---

## View

```html
<input type="file"
       name="Files"
       multiple />
```

---

## Controller

```csharp
[HttpPost]
public async Task<IActionResult> UploadMultiple(
    MultipleFileUploadViewModel model)
{
    foreach (var file in model.Files)
    {
        if (file.Length > 0)
        {
            string path =
                Path.Combine(
                    _environment.WebRootPath,
                    "uploads",
                    file.FileName);

            using var stream =
                new FileStream(
                    path,
                    FileMode.Create);

            await file.CopyToAsync(stream);
        }
    }

    return View();
}
```

# File Validation

Always validate files before saving.

## File Size Validation

```csharp
if (file.Length > 5 * 1024 * 1024)
{
    ModelState.AddModelError(
        "",
        "Maximum file size is 5 MB");
}
```

---

## Extension Validation

```csharp
var allowedExtensions =
    new[] { ".jpg", ".png", ".pdf" };

string extension =
    Path.GetExtension(
        file.FileName);

if (!allowedExtensions.Contains(
        extension.ToLower()))
{
    ModelState.AddModelError(
        "",
        "Invalid File Type");
}
```

---

## Content Type Validation

```csharp
var allowedTypes =
    new[]
    {
        "image/jpeg",
        "image/png",
        "application/pdf"
    };

if (!allowedTypes.Contains(
        file.ContentType))
{
    ModelState.AddModelError(
        "",
        "Invalid File Format");
}
```

# Preventing File Name Conflicts

If two users upload the same file name:

```text
resume.pdf
resume.pdf
```

Second file overwrites first file.

Use a unique file name:

```csharp
string uniqueFileName =
    Guid.NewGuid() +
    Path.GetExtension(
        file.FileName);
```

Example:

```text
c0f5b94e-a4d2.pdf
```

# Downloading Files

## Controller

```csharp
public IActionResult Download(
    string fileName)
{
    string filePath =
        Path.Combine(
            _environment.WebRootPath,
            "uploads",
            fileName);

    byte[] fileBytes =
        System.IO.File.ReadAllBytes(
            filePath);

    return File(
        fileBytes,
        "application/octet-stream",
        fileName);
}
```

# Uploading Files to Database

Instead of storing files physically, store bytes.

## Entity

```csharp
public class Document
{
    public int Id { get; set; }

    public string FileName { get; set; }

    public byte[] Data { get; set; }

```

---

## Save File

```csharp
using var memoryStream =
    new MemoryStream();

await file.CopyToAsync(
    memoryStream);

var document =
    new Document
    {
        FileName = file.FileName,
        Data = memoryStream.ToArray()
    };

_context.Documents.Add(document);

await _context.SaveChangesAsync();
```

# Using Azure Blob Storage

For cloud applications, files are often stored in:

* Azure Blob Storage
* AWS S3
* Google Cloud Storage

Advantages:

* Scalable
* Secure
* CDN support
* Lower server storage usage

Since you've already worked with Azure Blob Storage, this is the preferred approach for production applications rather than storing files inside `wwwroot`.

# Interview Questions

### What is IFormFile?

`IFormFile` represents a file sent with an HTTP request and is used for file uploads in ASP.NET Core MVC.

---

### Why is `multipart/form-data` required?

It allows form data and file content to be transmitted together in an HTTP request.

---

### How do you upload multiple files?

Use:

```csharp
List<IFormFile>
```

and add the `multiple` attribute to the file input.

---

### How do you avoid duplicate file names?

Generate unique names using:

```csharp
Guid.NewGuid()
```

---

### Which is better: Database or File System?

* Small files → Database can be acceptable.
* Large files → File System or Azure Blob Storage is preferred.

---

### What is IWebHostEnvironment used for?

It provides access to application paths such as:

```csharp
_environment.WebRootPath
```

which points to the `wwwroot` folder where uploaded files are commonly stored.
