# How to Import Excel Data to Database in ASP.NET Core MVC

Importing Excel data into a database is a common requirement in:

* Employee Management Systems
* Hospital Management Systems
* Inventory Systems
* Restaurant Applications
* Travel Applications

Typical flow:

```text
Excel File (.xlsx)
        ↓
Upload File
        ↓
Read Excel Data
        ↓
Validate Data
        ↓
Map to Entity
        ↓
Save to Database
```

---

# Import Controller Action

```csharp
using OfficeOpenXml;

[HttpPost]
public async Task<IActionResult> ImportExcel(
    EmployeeImportViewModel model)
{
    if (model.File == null ||
        model.File.Length == 0)
    {
        ModelState.AddModelError(
            "",
            "Please select an Excel file.");

        return View();
    }

    ExcelPackage.LicenseContext =
        LicenseContext.NonCommercial;

    using var stream =
        new MemoryStream();

    await model.File.CopyToAsync(stream);

    using var package =
        new ExcelPackage(stream);

    var worksheet =
        package.Workbook.Worksheets[0];

    int rowCount =
        worksheet.Dimension.Rows;

    var employees =
        new List<Employee>();

    for (int row = 2;
         row <= rowCount;
         row++)
    {
        employees.Add(
            new Employee
            {
                Name =
                    worksheet.Cells[row, 1]
                        .Text,

                Department =
                    worksheet.Cells[row, 2]
                        .Text,

                Salary =
                    decimal.Parse(
                        worksheet.Cells[row, 3]
                            .Text)
            });
    }

    _context.Employees.AddRange(
        employees);

    await _context.SaveChangesAsync();

    ViewBag.Message =
        "Data Imported Successfully";

    return View();
}
```

# Understanding the Code

## Copy File to Memory Stream

```csharp
using var stream =
    new MemoryStream();

await model.File.CopyToAsync(
    stream);
```

Loads the uploaded Excel file into memory.

---

## Open Excel Workbook

```csharp
using var package =
    new ExcelPackage(stream);
```

Creates an Excel workbook object.

---

## Access Worksheet

```csharp
var worksheet =
    package.Workbook.Worksheets[0];
```

Gets the first worksheet.

---

## Get Row Count

```csharp
int rowCount =
    worksheet.Dimension.Rows;
```

Returns:

```text
Total rows in worksheet
```

---

## Skip Header Row

```csharp
for (int row = 2;
     row <= rowCount;
     row++)
```

Because row 1 contains:

```text
Name
Department
Salary
```

---

## Read Cell Values

```csharp
worksheet.Cells[row,1].Text
```

Reads column 1 value.

Example:

```text
John
```

# Validating Excel File Type

Before importing:

```csharp
string extension =
    Path.GetExtension(
        model.File.FileName);

if (extension != ".xlsx")
{
    ModelState.AddModelError(
        "",
        "Only Excel files are allowed.");

    return View();
}
```

---

# Handling Parsing Errors

Instead of:

```csharp
decimal.Parse(...)
```

Use:

```csharp
decimal salary;

decimal.TryParse(
    worksheet.Cells[row,3].Text,
    out salary);
```

Safer for invalid data.

---

# Row-Level Validation

```csharp
string name =
    worksheet.Cells[row,1].Text;

if (string.IsNullOrWhiteSpace(name))
{
    continue;
}
```

Skip empty rows.

---

# Prevent Duplicate Records

Example:

```csharp
bool exists =
    await _context.Employees
        .AnyAsync(x =>
            x.Name == name);

if (exists)
{
    continue;
}
```

Avoid importing duplicates.

* Easier maintenance

# Bulk Insert Optimization

For small files:

```csharp
_context.Employees.AddRange(
    employees);

await _context.SaveChangesAsync();
```

Works well.

---

For large files (thousands of rows):

Use:

* Bulk Extensions
* SQL Bulk Copy
* Batch Processing

```text

```

This greatly improves performance.

# Real-World Service-Based Approach

Instead of putting everything in the controller:

```text
Controller
    ↓
ExcelImportService
    ↓
Repository/DbContext
```

Example:

```csharp
public interface IExcelImportService
{
    Task ImportEmployeesAsync(
        IFormFile file);
}
```

Controller:

```csharp
await _excelImportService
    .ImportEmployeesAsync(
        model.File);
```

This follows the Single Responsibility Principle (SRP).

# Interview Questions

### Which library is commonly used for Excel import in ASP.NET Core MVC?

EPPlus, ClosedXML, and OpenXML SDK.

---

### Why use MemoryStream during import?

To load the uploaded file into memory and allow EPPlus to read it.

---

### Why start reading from row 2?

Row 1 usually contains column headers.

---

### Why use TryParse instead of Parse?

`TryParse()` prevents exceptions when the data is invalid.

---

### How do you prevent duplicate imports?

Check the database using:

```csharp
AnyAsync()
```

before inserting records.

---

### What is the recommended architecture for Excel import?

```text
Controller
    ↓
Service Layer
    ↓
Repository/DbContext
```

This keeps controllers thin, improves testability, and follows clean architecture principles.
