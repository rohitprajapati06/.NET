# Working with Bar Chart and PDF in ASP.NET Core MVC

A common business requirement is to:

1. Fetch data from the database
2. Generate a chart (Sales, Orders, Revenue, etc.)
3. Embed the chart into a PDF report
4. Download or email the PDF

Example:

```text
Database Data
      ↓
Generate Bar Chart
      ↓
Convert Chart to Image
      ↓
Insert Image into PDF
      ↓
Download PDF
```

Common use cases:

* Sales Reports
* Employee Performance Reports
* Restaurant Revenue Reports
* Hospital Statistics Reports
* Inventory Reports

---

# Generate Bar Chart Image

```csharp
using ScottPlot;

public byte[] GenerateSalesChart(
    List<SalesReportDto> sales)
{
    var plot = new Plot();

    double[] values =
        sales.Select(x => (double)x.Sales)
             .ToArray();

    string[] labels =
        sales.Select(x => x.Month)
             .ToArray();

    plot.Add.Bars(values);

    plot.Axes.Bottom.TickGenerator =
        new ScottPlot.TickGenerators.NumericManual(
            Enumerable.Range(0, labels.Length)
                      .Select(x => (double)x)
                      .ToArray(),
            labels);

    return plot.GetImageBytes(
        800,
        500,
        ScottPlot.ImageFormat.Png);
}
```

Output:

```text
Bar Chart Image (PNG)
```

# Generate PDF with Chart

```csharp
public byte[] GeneratePdf(
    byte[] chartImage)
{
    return Document.Create(container =>
    {
        container.Page(page =>
        {
            page.Margin(20);

            page.Header()
                .Text("Monthly Sales Report")
                .FontSize(20)
                .Bold();

            page.Content()
                .Column(column =>
                {
                    column.Item()
                        .Text(
                            "Sales Performance");

                    column.Item()
                        .Image(chartImage);
                });

            page.Footer()
                .AlignCenter()
                .Text("Generated by ASP.NET Core");
        });
    })
    .GeneratePdf();
}
```

# Controller Action

```csharp
public IActionResult DownloadSalesReport()
{
    var sales =
        new List<SalesReportDto>
        {
            new() { Month = "Jan", Sales = 50000 },
            new() { Month = "Feb", Sales = 65000 },
            new() { Month = "Mar", Sales = 70000 },
            new() { Month = "Apr", Sales = 90000 }
        };

    byte[] chart =
        _reportService
            .GenerateSalesChart(sales);

    byte[] pdf =
        _reportService
            .GeneratePdf(chart);

    return File(
        pdf,
        "application/pdf",
        "SalesReport.pdf");
}
```

# Database Driven Example

```csharp
var sales =
    await _context.Orders
        .GroupBy(x => x.OrderDate.Month)
        .Select(g => new SalesReportDto
        {
            Month = g.Key.ToString(),
            Sales = g.Sum(x => x.TotalAmount)
        })
        .ToListAsync();
```

This automatically generates charts from actual database data.

# Interview Questions

### How do you add a chart to a PDF in ASP.NET Core MVC?

Generate the chart as an image (PNG/JPG) and embed it into the PDF using QuestPDF or iText7.

### Which libraries are commonly used?

* Chart: ScottPlot, ChartJS, OxyPlot
* PDF: QuestPDF, iText7

### Why convert charts to images?

PDF libraries cannot directly render chart objects; they typically embed image bytes.

### Where should report generation logic reside?

In a dedicated service such as:

```text
IReportService
    ↓
ReportService
```

rather than inside controller actions.

### What is a common enterprise reporting workflow?

```text
Database
    ↓
Aggregate Data
    ↓
Generate Charts
    ↓
Generate PDF
    ↓
Store / Download / Email
```

This pattern is commonly used in ERP, healthcare, restaurant, banking, and analytics systems.
