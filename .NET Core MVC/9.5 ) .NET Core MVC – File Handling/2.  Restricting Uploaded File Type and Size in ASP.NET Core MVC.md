# Restricting Uploaded File Type and Size in ASP.NET Core MVC

Restricting uploaded files is an important security measure in ASP.NET Core MVC. Without validation, users may upload:

* Executable files (`.exe`, `.bat`)
* Malicious scripts (`.js`, `.php`)
* Extremely large files causing storage issues
* Unsupported file formats

Therefore, always validate:

1. **File Extension**
2. **Content Type (MIME Type)**
3. **File Size**

---

# Why Restrict File Uploads?

Suppose your application only accepts:

```text
.jpg
.jpeg
.png
.pdf
```

A user could try uploading:

```text
virus.exe
malware.bat
script.js
```

Without validation, these files might be stored on the server and create security risks.

# Restricting MIME Types

Checking extensions alone is not enough.

A malicious user can rename:

```text
virus.exe
```

to

```text
virus.jpg
```

Therefore, verify the MIME type as well.

---

## Allowed MIME Types

```csharp
private readonly string[] _allowedContentTypes =
{
    "image/jpeg",
    "image/png",
    "application/pdf"
};
```

---

## Validation

```csharp
if (!_allowedContentTypes.Contains(
        file.ContentType))
{
    ModelState.AddModelError(
        "",
        "Invalid file type.");
}
```

# Restricting File Size

Large files can:

* Consume storage
* Slow uploads
* Cause denial-of-service issues

---

## Example: Maximum 5 MB

```csharp
const long maxFileSize =
    5 * 1024 * 1024;

if (file.Length > maxFileSize)
{
    ModelState.AddModelError(
        "",
        "Maximum file size is 5 MB.");
}
```

---

| Size  | Calculation      |
| ----- | ---------------- |
| 1 KB  | 1024 Bytes       |
| 1 MB  | 1024 × 1024      |
| 5 MB  | 5 × 1024 × 1024  |
| 10 MB | 10 × 1024 × 1024 |

# Complete Validation Example

```csharp
[HttpPost]
public async Task<IActionResult> Upload(
    FileUploadViewModel model)
{
    if (model.File == null)
    {
        ModelState.AddModelError(
            "",
            "Please select a file.");

        return View();
    }

    string[] allowedExtensions =
    {
        ".jpg",
        ".jpeg",
        ".png",
        ".pdf"
    };

    string[] allowedContentTypes =
    {
        "image/jpeg",
        "image/png",
        "application/pdf"
    };

    const long maxFileSize =
        5 * 1024 * 1024;

    string extension =
        Path.GetExtension(
            model.File.FileName)
            .ToLowerInvariant();

    if (!allowedExtensions.Contains(
            extension))
    {
        ModelState.AddModelError(
            "",
            "Invalid file extension.");
    }

    if (!allowedContentTypes.Contains(
            model.File.ContentType))
    {
        ModelState.AddModelError(
            "",
            "Invalid content type.");
    }

    if (model.File.Length >
        maxFileSize)
    {
        ModelState.AddModelError(
            "",
            "File size exceeds 5 MB.");
    }

    if (!ModelState.IsValid)
    {
        return View();
    }

    string uploadsFolder =
        Path.Combine(
            _environment.WebRootPath,
            "uploads");

    Directory.CreateDirectory(
        uploadsFolder);

    string fileName =
        Guid.NewGuid() +
        extension;

    string filePath =
        Path.Combine(
            uploadsFolder,
            fileName);

    using var stream =
        new FileStream(
            filePath,
            FileMode.Create);

    await model.File.CopyToAsync(
        stream);

    ViewBag.Message =
        "File uploaded successfully.";

    return View();
}
```

# Creating a Custom Validation Attribute

Instead of repeating validation logic, create a reusable attribute.

## Custom Attribute

```csharp
public class AllowedExtensionsAttribute
    : ValidationAttribute
{
    private readonly string[] _extensions;

    public AllowedExtensionsAttribute(
        string[] extensions)
    {
        _extensions = extensions;
    }

    protected override ValidationResult IsValid(
        object value,
        ValidationContext validationContext)
    {
        var file = value as IFormFile;

        if (file != null)
        {
            string extension =
                Path.GetExtension(
                    file.FileName);

            if (!_extensions.Contains(
                    extension.ToLower()))
            {
                return new ValidationResult(
                    $"Allowed extensions: {string.Join(", ", _extensions)}");
            }
        }

        return ValidationResult.Success;
    }
}
```

---

## Use in Model

```csharp

{
    [AllowedExtensions(
        new[] { ".jpg", ".png", ".pdf" })]
    public IFormFile File { get; set; }
}
```

Instead of:



Avoid:

```csharp

```

Use:

```csharp
Guid.NewGuid()
```

---

### ✔ Scan Files for Malware

Use antivirus scanning before saving files.

# Interview Questions

### Why validate file extensions?

To prevent users from uploading unsupported or potentially dangerous file types.

---

### Why check MIME type if extension is already checked?

Because users can rename malicious files (e.g., `virus.exe` → `virus.jpg`). MIME type validation adds another layer of protection.

---

### How do you limit upload size in ASP.NET Core MVC?

Using:

```csharp
[RequestSizeLimit]
```

or configuring:

```csharp
IISServerOptions.MaxRequestBodySize
```

---

### Which is safer: extension validation or MIME type validation?

Neither alone is sufficient. Use both together, and for high-security systems also inspect the file's binary signature (magic numbers).

---

### What is the recommended approach for production?

* Validate extension
* Validate MIME type
* Validate file size
* Generate unique file names
* Store files outside `wwwroot` when possible
* Scan uploaded files for malware
* Use cloud storage such as Azure Blob Storage for scalability and security.
