# How to Export Data to Excel File in ASP.NET Core MVC

Exporting data to Excel is a common requirement in business applications such as:

* Employee Management Systems
* Inventory Systems
* Restaurant Management Systems
* Healthcare Systems
* Reporting Dashboards

In ASP.NET Core MVC, the most popular library for Excel export is **[EPPlus](https://epplussoftware.com/?utm_source=chatgpt.com)**.

---

# Steps to Export Data to Excel

1. Install EPPlus Package
2. Fetch Data from Database
3. Create Excel Workbook
4. Add Worksheet and Data
5. Return Excel File to Browser

---

# Install EPPlus

Using Package Manager Console:

```powershell
Install-Package EPPlus
```

Or .NET CLI:

```bash
dotnet add package EPPlus
```

# Controller Action for Excel Export

```csharp
using OfficeOpenXml;

public class EmployeeController : Controller
{
    private readonly ApplicationDbContext _context;

    public EmployeeController(
        ApplicationDbContext context)
    {
        _context = context;
    }

    public async Task<IActionResult> ExportToExcel()
    {
        var employees =
            await _context.Employees
                .ToListAsync();

        ExcelPackage.LicenseContext =
            LicenseContext.NonCommercial;

        using var package =
            new ExcelPackage();

        var worksheet =
            package.Workbook.Worksheets
                .Add("Employees");

        worksheet.Cells[1, 1].Value = "Id";
        worksheet.Cells[1, 2].Value = "Name";
        worksheet.Cells[1, 3].Value = "Department";
        worksheet.Cells[1, 4].Value = "Salary";

        int row = 2;

        foreach (var employee in employees)
        {
            worksheet.Cells[row, 1].Value =
                employee.Id;

            worksheet.Cells[row, 2].Value =
                employee.Name;

            worksheet.Cells[row, 3].Value =
                employee.Department;

            worksheet.Cells[row, 4].Value =
                employee.Salary;

            row++;
        }

        worksheet.Cells.AutoFitColumns();

        var excelBytes =
            package.GetAsByteArray();

        return File(
            excelBytes,
            "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
            "Employees.xlsx");
    }
}
```

# Explanation

## Create Workbook

```csharp
using var package =
    new ExcelPackage();
```

Creates a new Excel workbook in memory.

---

## Add Worksheet

```csharp
var worksheet =
    package.Workbook.Worksheets
        .Add("Employees");
```

Creates a worksheet named:

```text
Employees
```

---

## Add Headers

```csharp
worksheet.Cells[1, 1].Value = "Id";
worksheet.Cells[1, 2].Value = "Name";
worksheet.Cells[1, 3].Value = "Department";
worksheet.Cells[1, 4].Value = "Salary";
```

Produces:

| Id | Name | Department | Salary |
| -- | ---- | ---------- | ------ |

---

## Populate Rows

```csharp
foreach (var employee in employees)
{
    ...
}
```

Adds data row by row.

---

## Auto-Fit Columns

```csharp
worksheet.Cells.AutoFitColumns();
```

Adjusts column widths automatically.

---

## Return File

```csharp
return File(
    excelBytes,
    "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
```

Browser downloads:

```csharp
worksheet.Cells["A1:D1"]
```

## Background Color

```csharp
using System.Drawing;

worksheet.Cells["A1:D1"]
```

# Export Data from DTO

This is the preferred approach.

```csharp
public class EmployeeExportDto
{
    public string Name { get; set; }

    public string Department { get; set; }
}
```

```csharp
var data =
    await _context.Employees
        .Select(e => new EmployeeExportDto
        {
            Name = e.Name,
            Department = e.Department
        })
        .ToListAsync();
```

Benefits:

* Better performance
* Less data transferred
* Cleaner code

# Export Large Datasets

For very large exports:

```csharp
var employees =
    await _context.Employees
        .AsNoTracking()
        .ToListAsync();
```

Benefits:

* Faster query execution
* Lower memory usage
* Better export performance

# Alternative Libraries

| Library     | Description                  |
| ----------- | ---------------------------- |
| EPPlus      | Most popular, feature-rich   |
| ClosedXML   | Easier API, built on OpenXML |
| OpenXML SDK | Microsoft's official library |
| NPOI        | Java POI port for .NET       |

Many developers prefer **[ClosedXML](https://closedxml.io/?utm_source=chatgpt.com)** for simpler syntax, while EPPlus offers more advanced Excel features.

# Best Practices

### ✔ Use DTOs

```csharp
EmployeeExportDto
```

instead of exporting entities directly.

### ✔ Use AsNoTracking()

```csharp
.AsNoTracking()
```

for read-only exports.

### ✔ Auto-Fit Columns

```csharp
worksheet.Cells.AutoFitColumns();
```

### ✔ Style Headers

Make reports easier to read.

### ✔ Use Meaningful File Names

```csharp
Employees_20260814.xlsx
```

### ✔ Export Only Necessary Data

Avoid exporting sensitive information.

# Interview Questions

### Which library is commonly used for Excel export in ASP.NET Core MVC?

EPPlus and ClosedXML are the most commonly used libraries.

### What MIME type is used for Excel files?

```text
application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
```

### Why use DTOs for export?

To export only required fields and improve performance.

### What does AutoFitColumns() do?

It automatically adjusts column widths based on content.

### Why use AsNoTracking() during export?

Because the data is read-only and doesn't need Entity Framework change tracking, which improves performance.
