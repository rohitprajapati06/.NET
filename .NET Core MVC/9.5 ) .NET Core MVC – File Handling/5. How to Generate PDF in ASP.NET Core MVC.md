# How to Generate PDF in ASP.NET Core MVC

PDF generation is commonly used for:

* Invoices
* Reports
* Receipts
* Certificates
* Order Summaries
* Medical Reports
* Booking Confirmations

Typical flow:

```text
Database Data
      ↓
Controller
      ↓
Generate PDF
      ↓
Download/Open PDF
```

---

# Popular PDF Libraries in ASP.NET Core MVC

| Library             | Description                    |
| ------------------- | ------------------------------ |
| iText7              | Most powerful and feature-rich |
| QuestPDF            | Modern and developer-friendly  |
| DinkToPdf           | Converts HTML to PDF           |
| Rotativa.AspNetCore | Converts Razor Views to PDF    |
| PdfSharpCore        | Basic PDF generation           |

For modern ASP.NET Core applications, **QuestPDF** is widely preferred due to its clean API.

# Basic PDF Generation

## Controller Action

```csharp
using QuestPDF.Fluent;
using QuestPDF.Helpers;
using QuestPDF.Infrastructure;

public IActionResult GeneratePdf()
{
    var pdf = Document.Create(container =>
    {
        container.Page(page =>
        {
            page.Margin(30);

            page.Header()
                .Text("Employee Report")
                .FontSize(20)
                .Bold();

            page.Content()
                .Text("This is a sample PDF generated using QuestPDF.");

            page.Footer()
                .AlignCenter()
                .Text(x =>
                {
                    x.Span("Page ");
                    x.CurrentPageNumber();
                });
        });
    });

    var pdfBytes = pdf.GeneratePdf();

    return File(
        pdfBytes,
        "application/pdf",
        "EmployeeReport.pdf");
}
```

# Create Employee Report PDF

```csharp
public async Task<IActionResult> EmployeeReport()
{
    var employees = await _context.Employees
        .AsNoTracking()
        .ToListAsync();

    var pdf = Document.Create(container =>
    {
        container.Page(page =>
        {
            page.Margin(20);

            page.Header()
                .Text("Employee Report")
                .FontSize(20)
                .Bold();

            page.Content()
                .Table(table =>
                {
                    table.ColumnsDefinition(columns =>
                    {
                        columns.RelativeColumn();
                        columns.RelativeColumn();
                        columns.RelativeColumn();
                    });

                    table.Header(header =>
                    {
                        header.Cell().Text("Name");
                        header.Cell().Text("Department");
                        header.Cell().Text("Salary");
                    });

                    foreach (var employee in employees)
                    {
                        table.Cell().Text(employee.Name);
                        table.Cell().Text(employee.Department);
                        table.Cell().Text(
                            employee.Salary.ToString("C"));
                    }
                });
        });
    });

    return File(
        pdf.GeneratePdf(),
        "application/pdf",
        "Employees.pdf");
}
```

# Generate PDF Using DTO

Recommended approach:

```csharp
public class EmployeeReportDto
{
    public string Name { get; set; }

    public string Department { get; set; }

    public decimal Salary { get; set; }
}
```

```csharp
var data = await _context.Employees
    .Select(x => new EmployeeReportDto
    {
        Name = x.Name,
        Department = x.Department,
        Salary = x.Salary
    })
    .ToListAsync();
```

Benefits:

* Better performance
* Less data fetched
* Cleaner design

# Generate PDF from Razor View

Another approach is converting a Razor View to PDF using **[Rotativa.AspNetCore](https://github.com/webgio/Rotativa.AspNetCore?utm_source=chatgpt.com)**.

```csharp
public IActionResult Invoice()
{
    return new ViewAsPdf("Invoice");
}
```

Advantages:

* Reuse existing Razor views
* Easy styling with Bootstrap
* Faster development

# Best Practices

### ✔ Use DTOs

```csharp
EmployeeReportDto
```

instead of entities.

---

### ✔ Use AsNoTracking()

```csharp
.AsNoTracking()
```

for reports.

---

### ✔ Generate PDFs in Service Layer

```text
Controller
    ↓
PdfService
    ↓
QuestPDF
```

Avoid putting PDF logic in controllers.

---

### ✔ Add Branding

Include:

* Logo
* Company Name
* Footer
* Contact Information

---

### ✔ Handle Large Reports Carefully

For thousands of records:

* Use pagination
* Stream data
* Generate in background jobs if necessary

---

### ✔ Use Consistent Layouts

Create reusable templates for:

* Invoices
* Reports
* Receipts
* Certificates
