# How to Generate Password-Protected PDF in ASP.NET Core MVC

A password-protected PDF requires users to enter a password before opening the document.

Common use cases:

* Salary Slips
* Medical Reports
* Bank Statements
* Invoices
* Legal Documents
* Employee Records

Example:

```text
SalarySlip.pdf
        ↓
Enter Password
        ↓
View PDF Content
```

---

# Libraries That Support PDF Encryption

| Library      | Password Protection Support    |
| ------------ | ------------------------------ |
| iText7       | ✔ Excellent                    |
| QuestPDF     | ✖ No native encryption         |
| PdfSharpCore | Limited                        |
| DinkToPdf    | Depends on PDF post-processing |

Since **QuestPDF does not currently provide built-in PDF password encryption**, the most common approach is:

```text
Generate PDF (QuestPDF/iText7)
            ↓
Encrypt PDF using iText7
            ↓
Return Protected PDF
```

For password-protected PDFs, **iText7** is generally the preferred solution.

# Basic Password-Protected PDF Example

## Controller Action

```csharp
using iText.Kernel.Pdf;
using iText.Layout;
using iText.Layout.Element;

public IActionResult GenerateProtectedPdf()
{
    using var memoryStream =
        new MemoryStream();

    string password = "Admin@123";

    var writerProperties =
        new WriterProperties()
            .SetStandardEncryption(
                System.Text.Encoding.UTF8.GetBytes(password),
                System.Text.Encoding.UTF8.GetBytes(password),
                EncryptionConstants.ALLOW_PRINTING,
                EncryptionConstants.ENCRYPTION_AES_256);

    var writer =
        new PdfWriter(
            memoryStream,
            writerProperties);

    var pdf =
        new PdfDocument(writer);

    var document =
        new Document(pdf);

    document.Add(
        new Paragraph(
            "Confidential Employee Report"));

    document.Close();

    return File(
        memoryStream.ToArray(),
        "application/pdf",
        "ProtectedReport.pdf");
}
```

## Encryption


# Using Different User and Owner Passwords

A PDF can have:

1. User Password
2. Owner Password

### User Password

Required to open the PDF.

### Owner Password

Controls permissions.

Example:

```csharp
string userPassword =
    "Employee123";

string ownerPassword =
    "AdminSecret123";
```

```csharp
.SetStandardEncryption(
    Encoding.UTF8.GetBytes(userPassword),
    Encoding.UTF8.GetBytes(ownerPassword),
    EncryptionConstants.ALLOW_PRINTING,
    EncryptionConstants.ENCRYPTION_AES_256);
```

# Example: Employee Salary Slip

```csharp
public IActionResult SalarySlip()
{
    string employeeId = "EMP001";

    string password =
        employeeId;

    using var stream =
        new MemoryStream();

    var writerProps =
        new WriterProperties()
            .SetStandardEncryption(
                Encoding.UTF8.GetBytes(password),
                Encoding.UTF8.GetBytes("AdminSecret"),
                EncryptionConstants.ALLOW_PRINTING,
                EncryptionConstants.ENCRYPTION_AES_256);

    var writer =
        new PdfWriter(
            stream,
            writerProps);

    var pdf =
        new PdfDocument(writer);

    var document =
        new Document(pdf);

    document.Add(
        new Paragraph(
            "Salary Slip"));

    document.Add(
        new Paragraph(
            "Employee ID: EMP001"));

    document.Add(
        new Paragraph(
            "Salary: ₹75,000"));

    document.Close();

    return File(
        stream.ToArray(),
        "application/pdf",
        "SalarySlip.pdf");
}
```

Opening the PDF requires:

```text
EMP001
```

as the password.

# Generate PDF First, Encrypt Later

Sometimes you already have a PDF.

Workflow:

```text
Generate PDF
      ↓
Save PDF
      ↓
Encrypt Existing PDF
      ↓
Return Protected PDF
```

Example:

```csharp
var reader =
    new PdfReader(sourcePdfPath);

var writer =
    new PdfWriter(
        destinationPdfPath,
        writerProperties);

var pdf =
    new PdfDocument(
        reader,
        writer);

pdf.Close();
```

This applies encryption to an existing PDF.

# Protecting QuestPDF Output

Generate PDF with QuestPDF:

```csharp
byte[] pdfBytes =
    document.GeneratePdf();
```

Then encrypt:

```csharp
using var input =
    new MemoryStream(pdfBytes);

using var output =
    new MemoryStream();

var reader =
    new PdfReader(input);

var writer =
    new PdfWriter(
        output,
        writerProperties);

var pdf =
    new PdfDocument(
        reader,
        writer);

pdf.Close();

return File(
    output.ToArray(),
    "application/pdf",
    "Protected.pdf");
```

This combines:

```text
QuestPDF
     +
iText7 Encryption
```

# Security Considerations

### Avoid Hardcoded Passwords

Bad:

```csharp
string password =
    "Admin123";
```

Good:

```csharp
string password =
    employee.EmployeeCode;
```

or

```csharp
string password =
    GenerateRandomPassword();
```

---

### Use Strong Encryption

Prefer:

```csharp
EncryptionConstants.ENCRYPTION_AES_256
```

Avoid older encryption standards.

---

### Use Different Owner Password

```csharp
Owner Password
≠
User Password
```

Provides better control over document permissions.

---

### Store Password Securely

If passwords are generated dynamically:

* Store securely
* Do not log them
* Avoid exposing them in URLs

# Service-Based Architecture

```text
Controller
    ↓
PdfService
    ↓
EncryptionService
    ↓
iText7
```

Example:

```csharp
public interface IPdfService
{
    byte[] GenerateProtectedPdf(
        EmployeeDto employee,
        string password);
}
```

Controller:

```csharp
var pdf =
    _pdfService.GenerateProtectedPdf(
        employee,
        password);
```

This keeps controllers clean and improves testability.

# Interview Questions

### Can QuestPDF generate password-protected PDFs directly?

No. QuestPDF generates PDFs but does not currently provide native password encryption. A common approach is to generate the PDF and then encrypt it using iText7.

---

### What is the difference between a user password and an owner password?

* **User Password**: Required to open the PDF.
* **Owner Password**: Controls permissions such as printing, copying, and editing.

---

### Which encryption algorithm is recommended?

```csharp
EncryptionConstants.ENCRYPTION_AES_256
```

because it provides strong modern encryption.

---

### How do you return a protected PDF from an ASP.NET Core MVC controller?

```csharp
return File(
    pdfBytes,
    "application/pdf",
    "ProtectedReport.pdf");
```

where `pdfBytes` contains the encrypted PDF content.

---

### Where should PDF generation logic be placed?

In a dedicated service such as:

```text
IPdfService
    ↓
PdfService
```

rather than directly inside controller actions.
