# Middleware in ASP.NET Core Web API

Middleware is a software component that is assembled into the application's request processing pipeline. Each middleware can:

* Process an incoming HTTP request.
* Perform some action before passing the request to the next middleware.
* Perform some action after the next middleware has executed.
* Short-circuit the pipeline and return a response directly.

## How Middleware Works

When a request arrives:

1. It enters the first middleware.
2. The middleware performs its work.
3. It calls the next middleware using `await next();`
4. After all middlewares execute, the response travels back through the pipeline in reverse order.

```text
Request
   │
   ▼
Middleware 1
   │
Middleware 2
   │
Middleware 3
   │
Controller
   │
Middleware 3
   │
Middleware 2
   │
Middleware 1
   │
Response
```

## Built-in Middleware

Some commonly used middleware in ASP.NET Core:

| Middleware              | Purpose                               |
| ----------------------- | ------------------------------------- |
| `UseRouting()`          | Matches request URLs to endpoints     |
| `UseAuthentication()`   | Identifies the user                   |
| `UseAuthorization()`    | Checks permissions                    |
| `UseExceptionHandler()` | Handles unhandled exceptions          |
| `UseHttpsRedirection()` | Redirects HTTP requests to HTTPS      |
| `UseCors()`             | Enables Cross-Origin Resource Sharing |
| `UseStaticFiles()`      | Serves static files                   |
| `UseSwagger()`          | Generates Swagger JSON                |
| `UseSwaggerUI()`        | Displays Swagger UI                   |

## Middleware Configuration Example

In `Program.cs`:

```csharp
var builder = WebApplication.CreateBuilder(args);

var app = builder.Build();

app.UseHttpsRedirection();

app.UseAuthentication();

app.UseAuthorization();

app.MapControllers();

app.Run();
```

Order matters because middleware executes in the order it is registered.

## Creating Custom Middleware

### Step 1: Create Middleware Class

```csharp
public class LoggingMiddleware
{
    private readonly RequestDelegate _next;

    public LoggingMiddleware(RequestDelegate next)
    {
        _next = next;
    }

    public async Task InvokeAsync(HttpContext context)
    {
        Console.WriteLine($"Request Path: {context.Request.Path}");

        await _next(context);

        Console.WriteLine($"Status Code: {context.Response.StatusCode}");
    }
}
```

---

### Step 2: Register Middleware

```csharp
app.UseMiddleware<LoggingMiddleware>();
```

## Middleware Execution Example

Suppose we have:

```csharp
app.UseMiddleware<FirstMiddleware>();
app.UseMiddleware<SecondMiddleware>();

app.MapControllers();
```

Request flow:

```text
Request enters FirstMiddleware

Before First

Request enters SecondMiddleware

Before Second

Controller Executes

After Second

After First

Response Sent
```

Output:

```text
Before First
Before Second
After Second
After First
```

## Inline Middleware

Instead of creating a class, middleware can be written inline.

```csharp
app.Use(async (context, next) =>
{
    Console.WriteLine("Before");

    await next();

    Console.WriteLine("After");
});
```

---

## `Use`, `Run`, and `Map`

### `Use`

Passes control to the next middleware.

```csharp
app.Use(async (context, next) =>
{
    await next();
});
```

### `Run`

Terminates the pipeline.

```csharp
app.Run(async context =>
{
    await context.Response.WriteAsync("Hello");
});
```

No middleware registered after `Run()` will execute.

### `Map`

Branches the pipeline based on a URL path.

```csharp
app.Map("/admin", adminApp =>
{
    adminApp.Run(async context =>
    {
        await context.Response.WriteAsync("Admin Area");
    });
});
```

## Middleware Order Best Practices

A typical order in a production Web API is:

```csharp
app.UseExceptionHandler("/error");

app.UseHttpsRedirection();

app.UseRouting();

app.UseCors();

app.UseAuthentication();

app.UseAuthorization();

app.MapControllers();
```

## Interview Questions

**Q1. What is middleware?**
Middleware is a component that handles HTTP requests and responses in the ASP.NET Core request pipeline.

**Q2. Why is middleware order important?**
Because requests move through middleware in registration order and responses move back in reverse order.

**Q3. Difference between `Use()` and `Run()`?**

| Use                      | Run                 |
| ------------------------ | ------------------- |
| Calls next middleware    | Terminates pipeline |
| `await next()` available | No next delegate    |

**Q4. When would you create custom middleware?**

Examples include:

* Global exception handling
* Request/response logging
* API key validation
* Correlation ID generation
* Custom authentication mechanisms
