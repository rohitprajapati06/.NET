# Introduction to ASP.NET Core Web API

## What is a Web API?

A **Web API (Application Programming Interface)** is a framework that allows applications to communicate with each other over HTTP.

For example:

```text
React Frontend
      |
      | HTTP Request
      v
ASP.NET Core Web API
      |
      | Database Query
      v
SQL Server Database
```

The frontend sends requests to the API, and the API processes the request, interacts with the database, and returns data in formats such as JSON.

---

## What is ASP.NET Core Web API?

**ASP.NET Core Web API** is a framework provided by Microsoft for building RESTful services using C# and .NET.

It is:

* Cross-platform (Windows, Linux, macOS)
* High-performance
* Open-source
* Cloud-ready
* Lightweight
* Suitable for Microservices and Enterprise Applications

---

# ASP.NET Core Web API Architecture

```text
Client
   |
   v
Controller
   |
   v
Service Layer
   |
   v
Repository Layer
   |
   v
Database
```

### Controller

Handles HTTP requests.

```csharp
[ApiController]
[Route("api/[controller]")]
public class UsersController : ControllerBase
{
}
```

### Service Layer

Contains business logic.

```csharp
public class UserService
{
}
```

### Repository Layer

Handles database operations.

```csharp
public class UserRepository
{
}
```

### Database

Stores application data.

Examples:

* SQL Server
* PostgreSQL
* MySQL

# REST API Basics

REST stands for:

**Representational State Transfer**

A REST API exposes resources through URLs.

Example:

```text
/api/users
```

Resource = Users

---

# HTTP Methods

| Method | Purpose                 |
| ------ | ----------------------- |
| GET    | Retrieve data           |
| POST   | Create data             |
| PUT    | Update entire resource  |
| PATCH  | Update partial resource |
| DELETE | Delete data             |

Example:

```http
GET /api/users
```

Returns:

```json
[
  {
    "id": 1,
    "name": "John"
  }
]
```

# Creating Your First Web API

## Step 1: Create Project

Using CLI:

```bash
dotnet new webapi -n DemoApi
```

Run:

```bash
cd DemoApi
dotnet run
```

---

## Step 2: Program.cs

In .NET 6+:

```csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();

builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

app.UseSwagger();
app.UseSwaggerUI();

app.MapControllers();

app.Run();
```

---

## Step 3: Create Controller

```csharp
using Microsoft.AspNetCore.Mvc;

namespace DemoApi.Controllers
{
    [ApiController]
    [Route("api/[controller]")]
    public class ProductsController : ControllerBase
    {
        [HttpGet]
        public IActionResult GetProducts()
        {
            return Ok("Products Retrieved");
        }
    }
}
```

# Action Results

ASP.NET Core provides standard HTTP responses.

### Ok (200)

```csharp
return Ok(data);
```

### Created (201)

```csharp
return CreatedAtAction(nameof(GetById), new { id }, product);
```

### BadRequest (400)

```csharp
return BadRequest("Invalid request");
```

### Unauthorized (401)

```csharp
return Unauthorized();
```

### NotFound (404)

```csharp
return NotFound();
```

### NoContent (204)

```csharp
return NoContent();
```

# Dependency Injection in Web API

Register service:

```csharp
builder.Services.AddScoped<IUserService, UserService>();
```

Inject into controller:

```csharp
public class UsersController : ControllerBase
{
    private readonly IUserService _userService;

    public UsersController(IUserService userService)
    {
        _userService = userService;
    }
}
```

Benefits:

* Loose coupling
* Easy testing
* Better maintainability

# Middleware Pipeline

Middleware processes requests sequentially.

```text
Request
   |
Authentication
   |
Authorization
   |
Controller
   |
Response
```

Example:

```csharp
app.UseAuthentication();
app.UseAuthorization();
```
