# Routing in ASP.NET Core Web API

Routing is the mechanism that matches an incoming HTTP request URL to a specific controller action.

For example, when a client sends:

```http
GET /api/hospitals/5
```

ASP.NET Core determines which controller and action method should handle that request.

---

# Request Processing Pipeline

When a request arrives:

```text
HTTP Request
      │
      ▼
Middleware Pipeline
      │
      ▼
Routing Middleware
      │
      ▼
Controller
      │
      ▼
Action Method
      │
      ▼
Response
```

In `.NET 6+`, routing is enabled automatically when using:

```csharp
app.MapControllers();
```

---

# Types of Routing

ASP.NET Core supports two routing approaches.

## 1. Attribute Routing (Most Common in APIs)

Routes are defined directly on controllers and actions.

Example:

```csharp
[ApiController]
[Route("api/[controller]")]
public class HospitalsController : ControllerBase
{
    [HttpGet]
    public IActionResult GetAll()
    {
        return Ok();
    }

    [HttpGet("{id}")]
    public IActionResult GetById(int id)
    {
        return Ok();
    }
}
```

### Generated Routes

| Method | URL              |
| ------ | ---------------- |
| GET    | /api/hospitals   |
| GET    | /api/hospitals/1 |

---

### `[controller]` token

```csharp
[Route("api/[controller]")]
```

ASP.NET Core replaces `[controller]` with the controller name minus `"Controller"`.

```text
HospitalsController
        ↓
hospitals
```

Result:

```text
api/hospitals
```

---

### `[action]` token

```csharp
[Route("api/[controller]/[action]")]
```

Controller:

```csharp
public class HospitalsController : ControllerBase
{
    [HttpGet]
    public IActionResult GetAll()
    {
        return Ok();
    }
}
```

Generated URL:

```text
/api/hospitals/GetAll
```

---

# 2. Conventional Routing

Mostly used in MVC applications.

Program.cs

```csharp
app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");
```

Example URL:

```text
/Home/Details/10
```

API projects generally prefer Attribute Routing.

# Route Parameters

## Single Parameter

```csharp
[HttpGet("{id}")]
public IActionResult Get(int id)
{
}
```

Request:

```text
GET /api/hospitals/10
```

Value received:

```text
id = 10
```

---

## Multiple Parameters

```csharp
[HttpGet("{hospitalId}/doctor/{doctorId}")]
public IActionResult GetDoctor(
        int hospitalId,
        int doctorId)
{
}
```

URL:

```text
GET /api/hospitals/5/doctor/20
```

Values:

```text
hospitalId = 5
doctorId = 20
```

---

# Optional Parameters

```csharp
[HttpGet("{id?}")]
public IActionResult Get(int? id)
{
}
```

Requests:

```text
/api/hospitals
```

or

```text
/api/hospitals/5
```

Both are valid.

---

# Route Constraints

Constraints ensure parameters have a specific format.

### Integer constraint

```csharp
[HttpGet("{id:int}")]
```

Only matches:

```text
/api/hospitals/10
```

Will not match:

```text
/api/hospitals/abc
```

---

### GUID constraint

Useful in your SmartHealthcare project since entities use `Guid`.

```csharp
[HttpGet("{id:guid}")]
public IActionResult Get(Guid id)
{
}
```

Example:

```text
GET /api/hospitals/3fa85f64-5717-4562-b3fc-2c963f66afa6
```

---

### Other constraints

```csharp
{id:min(1)}

{id:max(100)}

{name:alpha}

{age:range(18,60)}
```

# HTTP Method Attributes

Routing also considers the HTTP verb.

```csharp
[HttpGet]

[HttpPost]

[HttpPut]

[HttpDelete]

[HttpPatch]
```

Example:

```csharp
[HttpPost]
public IActionResult Create(CreateHospitalDto dto)
{
}
```

Request:

```text
POST /api/hospitals
```

---

# Named Routes

```csharp
[HttpGet("{id}", Name = "GetHospital")]
public IActionResult Get(Guid id)
{
}
```

Generate URL later:

```csharp
Url.RouteUrl("GetHospital",
    new { id = hospital.Id });
```

# How Routing Works in Your SmartHealthcare Project

Your controller looks similar to:

```csharp
[ApiController]
[Route("api/[controller]")]
public class HospitalsController : ControllerBase
{
    private readonly IMediator mediator;

    [HttpGet]
    public async Task<IActionResult> Get()
    {
        return Ok(await mediator.Send(
            new GetHospitalsQuery()));
    }

    [HttpGet("{id:guid}")]
    public async Task<IActionResult> Get(Guid id)
    {
        return Ok(await mediator.Send(
            new GetHospitalByIdQuery(id)));
    }
}
```

### Request Flow

```text
GET /api/hospitals

        │
        ▼

HospitalsController.Get()

        │
        ▼

Mediator.Send(GetHospitalsQuery)

        │
        ▼

GetHospitalsQueryHandler

        │
        ▼

Repository/DbContext

        │
        ▼

SQL Server

        │
        ▼

Hospital DTOs

        │
        ▼

HTTP 200 OK
```

# Best Practices for Web APIs

* Use **Attribute Routing**.
* Use route constraints such as `:guid`.
* Prefer plural resource names.

```text
/api/hospitals
/api/doctors
/api/patients
/api/appointments
```

* Avoid verbs in URLs.

Prefer:

```text
GET    /api/hospitals
POST   /api/hospitals
PUT    /api/hospitals/{id}
DELETE /api/hospitals/{id}
```

Instead of:

```text
/api/GetHospitals
/api/CreateHospital
/api/DeleteHospital
```

For interviews, a concise definition is:

> Routing in ASP.NET Core Web API is the process of matching an incoming HTTP request URL and HTTP method to a specific controller action that can handle the request.
