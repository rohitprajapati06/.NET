# EF Core — Saving Data

In Entity Framework Core, saving data means taking changes made in your entities (`Add`, `Update`, `Delete`) and persisting them into the database using `SaveChanges()` or `SaveChangesAsync()`.

---

# 1. Basic Flow of Saving Data

```text
Create/Modify Entity
        ↓
DbContext Tracks Changes
        ↓
SaveChanges()
        ↓
EF Core Generates SQL
        ↓
Database Updated
```

---

# 2. Insert Data (Create)

## Model

```csharp
public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
}
```

---

## DbContext

```csharp
public class AppDbContext : DbContext
{
    public DbSet<Product> Products { get; set; }

    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }
}
```

---

## Insert Single Record

```csharp
using var context = new AppDbContext(options);

var product = new Product
{
    Name = "Coffee",
    Price = 250
};

context.Products.Add(product);

context.SaveChanges();
```

---

# What Happens Internally?

Before `SaveChanges()`:

```text
Entity State = Added
```

EF Core generates SQL:

```sql
INSERT INTO Products (Name, Price)
VALUES ('Coffee', 250)
```

After saving:

```text
State = Unchanged
```

And `Id` gets populated automatically if it's an identity column.

---

# 3. Insert Multiple Records

```csharp
var products = new List<Product>
{
    new Product { Name = "Tea", Price = 100 },
    new Product { Name = "Burger", Price = 300 }
};

context.Products.AddRange(products);

context.SaveChanges();
```

---

# 4. Async Saving (Recommended)

In ASP.NET Core applications, prefer async operations.

```csharp
await context.Products.AddAsync(product);

await context.SaveChangesAsync();
```

---

# 5. Updating Data

## Step 1 — Fetch Data

```csharp
var product = context.Products.Find(1);
```

---

## Step 2 — Modify

```csharp
product.Price = 500;
```

---

## Step 3 — Save

```csharp
context.SaveChanges();
```

---

# Generated SQL

```sql
UPDATE Products
SET Price = 500
WHERE Id = 1
```

# Why `Update()` Is Not Required Here?

Because EF Core is already tracking the entity.

```text
Tracked Entity → EF Detects Changes Automatically
```

# 7. Entity States in EF Core

EF Core tracks entity states internally.

| State     | Meaning                    |
| --------- | -------------------------- |
| Added     | New entity to insert       |
| Modified  | Existing entity updated    |
| Deleted   | Entity marked for deletion |
| Unchanged | No changes                 |
| Detached  | Not tracked                |

---

# Example

```csharp
var product = new Product();

var state = context.Entry(product).State;
```

---

# 8. Change Tracker

EF Core keeps track of changes automatically.

```csharp
var product = context.Products.Find(1);

product.Price = 999;

context.ChangeTracker.DetectChanges();
```

Usually you do not call `DetectChanges()` manually.

---

# 9. Connected Scenario

Very important in real projects.

## Connected Scenario

```text
Same DbContext fetches and updates entity
```

Example:

```csharp
var product = context.Products.Find(1);

product.Price = 400;

context.SaveChanges();
```

EF already tracks the object.

---

# 10. Disconnected Scenario

Common in APIs.

```text
Data comes from client
New DbContext instance handles update
```

Example:

```csharp
[HttpPut]
public async Task<IActionResult> Update(Product product)
{
    _context.Products.Update(product);

    await _context.SaveChangesAsync();

    return Ok();
}
```

---

# Why `Update()` Needed Here?

Because EF Core did NOT fetch the entity earlier.

So EF must be told:

```text
"This entity is modified."
```

---

# 11. Add vs Attach vs Update

| Method   | State     |
| -------- | --------- |
| Add()    | Added     |
| Attach() | Unchanged |
| Update() | Modified  |
| Remove() | Deleted   |

---

# Example

```csharp
context.Attach(product);
```

EF tracks entity but assumes no changes.

---

# 12. SaveChanges() Transaction

`SaveChanges()` automatically uses a transaction.

Example:

```csharp
context.Products.Add(product1);
context.Products.Add(product2);

context.SaveChanges();
```

If one insert fails:

```text
Everything rolls back
```

---

# 13. Explicit Transactions

For complex operations.

```csharp
using var transaction = context.Database.BeginTransaction();

try
{
    context.Products.Add(product);

    context.SaveChanges();

    transaction.Commit();
}
catch
{
    transaction.Rollback();
}
```

# 14. Real ASP.NET Core MVC Example

public async Task<IActionResult> Create(Product product)

# 16. SaveChanges vs SaveChangesAsync

| SaveChanges     | SaveChangesAsync  |
| --------------- | ----------------- |
| Synchronous     | Asynchronous      |
| Blocks thread   | Non-blocking      |
| Console apps OK | Best for Web APIs |
| Less scalable   | More scalable     |

---

# 17. Common Mistakes

## Forgetting SaveChanges()

```csharp
context.Products.Add(product);
```

Without:

```csharp
context.SaveChanges();
```

Nothing gets stored.

---

## Using Update() on Tracked Entity

Unnecessary:

```csharp
var product = context.Products.Find(1);

product.Price = 100;

context.Products.Update(product);
```

Just modify + save.

---

## Multiple SaveChanges Calls

Bad:

```csharp
context.Add(a);
context.SaveChanges();

context.Add(b);
context.SaveChanges();
```

Better:

```csharp
context.Add(a);
context.Add(b);

context.SaveChanges();
```

# 18. Save Data Lifecycle

```text
Entity Created
    ↓
Tracked by DbContext
    ↓
State Changes
    ↓
SaveChanges()
    ↓
SQL Generated
    ↓
Database Updated
    ↓
State Reset
```

# 19. Interview Questions

## What does SaveChanges() do?

It detects tracked entity changes and generates SQL commands to persist data into the database.

---

## Difference between Add and Update?

| Add                | Update                  |
| ------------------ | ----------------------- |
| Inserts new record | Updates existing record |
| State = Added      | State = Modified        |

---

## What is Change Tracking?

Mechanism where EF Core monitors entity changes automatically.

---

## Why use SaveChangesAsync?

Improves scalability and avoids blocking threads in web applications.

---

# 20. Best Practices

✅ Use `async/await` in ASP.NET Core
✅ Use one DbContext per request
✅ Avoid unnecessary `Update()`
✅ Group multiple operations in one `SaveChanges()`
✅ Use transactions for critical operations
✅ Validate data before saving

# Real Project Example — La Caffeine

Example order save flow in your ASP.NET Core restaurant project:

```text
User Adds Items to Cart
        ↓
Create Order Entity
        ↓
Create OrderItems
        ↓
_context.Orders.Add(order)
        ↓
SaveChangesAsync()
        ↓
Order + Items saved together
```

Because of navigation properties and relationships, EF Core automatically inserts related child records.

Example:

```csharp
var order = new Order
{
    UserId = userId,
    TotalAmount = 1200,

    OrderItems = new List<OrderItem>
    {
        new OrderItem
        {
            ProductId = 1,
            Quantity = 2
        }
    }
};

_context.Orders.Add(order);

await _context.SaveChangesAsync();
```

EF Core inserts:

```text
Orders Table
OrderItems Table
```

in a single transaction automatically.
