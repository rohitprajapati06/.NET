# Entity Framework Core Installation

## What is EF Core?

Entity Framework Core is an ORM (Object Relational Mapper) used in ASP.NET Core applications to interact with databases using C# objects instead of writing raw SQL queries.

It supports:

- SQL Server
- MySQL
- PostgreSQL
- SQLite
- Cosmos DB
- InMemory Database

---

# Prerequisites

Before installing EF Core, ensure you have:

- .NET SDK installed
- Visual Studio / VS Code
- ASP.NET Core project created

Check installed SDK:

```bash
dotnet --version
```

---

# Step 1 — Create a Project

## ASP.NET Core MVC

```bash
dotnet new mvc -n EFCoreDemo
cd EFCoreDemo
```

## ASP.NET Core Web API

```bash
dotnet new webapi -n EFCoreDemo
cd EFCoreDemo
```

---

# Step 2 — Install EF Core Packages

EF Core works using NuGet packages.

---

# Important EF Core Packages

| Package | Purpose |
|---|---|
| Microsoft.EntityFrameworkCore | Core EF functionality |
| Microsoft.EntityFrameworkCore.SqlServer | SQL Server provider |
| Microsoft.EntityFrameworkCore.Tools | Migrations support |
| Microsoft.EntityFrameworkCore.Design | Design-time features |

---

# Installation Using .NET CLI

## Install Main EF Core Package

```bash
dotnet add package Microsoft.EntityFrameworkCore
```

---

## Install SQL Server Provider

```bash
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
```

---

## Install EF Core Tools

```bash
dotnet add package Microsoft.EntityFrameworkCore.Tools
```

---

## Install Design Package

```bash
dotnet add package Microsoft.EntityFrameworkCore.Design
```

# Step 3 — Install EF CLI Tools (Global)

This enables migration commands from terminal.

```bash
dotnet tool install --global dotnet-ef
```

Verify installation:

```bash
dotnet ef
```

---

# Step 4 — Verify Packages

Check installed packages:

```bash
dotnet list package
```

```text

Microsoft.EntityFrameworkCore.SqlServer
```

# Step 5 — Create First DbContext

Example:

```csharp
using Microsoft.EntityFrameworkCore;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options)
        : base(options)
    {
    }
}
```

---

# Step 6 — Register DbContext

## Program.cs (.NET 6/7/8)

```csharp
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(
        builder.Configuration.GetConnectionString("DefaultConnection")));
```

---

# Step 7 — Add Connection String

## appsettings.json

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=EFCoreDemoDb;Trusted_Connection=True;TrustServerCertificate=True"
  }
}
```

---

# Step 8 — Create First Migration

```bash
dotnet ef migrations add InitialCreate
```

This generates:

- Migrations folder
- Database schema code

---

# Step 9 — Update Database

```bash
dotnet ef database update
```

This creates database tables automatically.

# Beginner Mistakes to Avoid

## Forgetting Tools Package

Without this:

```bash
dotnet add package Microsoft.EntityFrameworkCore.Tools
```

Migration commands fail.

---

## Not Installing dotnet-ef

Without global tool:

```bash
dotnet ef migrations add InitialCreate
```

won’t work.

---

## Wrong Connection String

Most common issue.

For local SQL Server:

```text
Server=.;Database=YourDb;Trusted_Connection=True;TrustServerCertificate=True
```
