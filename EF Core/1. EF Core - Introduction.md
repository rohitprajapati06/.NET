# Introduction to Entity Framework Core

## What is EF Core?

**Entity Framework Core (EF Core)** is an Object Relational Mapper (ORM) for .NET.

It helps you work with databases using C# objects instead of writing large amounts of SQL manually.

Instead of:

```sql
SELECT * FROM Products
```

You can write:

```csharp
var products = await _context.Products.ToListAsync();
```

EF Core converts your C# code into SQL automatically.

---

# Why EF Core Exists

Without EF Core:

- You manually write SQL queries
- Open database connections
- Read data row by row
- Map data to objects manually

Example without ORM:

```csharp
SqlConnection con = new SqlConnection(connectionString);

SqlCommand cmd = new SqlCommand(
    "SELECT * FROM Products", con);

con.Open();

SqlDataReader reader = cmd.ExecuteReader();

while(reader.Read())
{
    Console.WriteLine(reader["Name"]);
}
```

This becomes repetitive and hard to maintain in large projects.

EF Core solves this by:

- Mapping database tables ↔ C# classes
- Generating SQL automatically
- Tracking changes
- Handling relationships
- Managing migrations

# Core Components of EF Core

## 1. Entity

A class representing a database table.

```csharp
public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
}
```

`Product` = Entity

---

## 2. DbContext

The bridge between your application and database.

```csharp
public class AppDbContext : DbContext
{
    public DbSet<Product> Products { get; set; }
}
```

### Responsibilities

- Database connection
- Query execution
- Change tracking
- Save operations

---

## 3. DbSet

Represents a database table.

```csharp
_context.Products
```

Equivalent to:

```sql
SELECT * FROM Products
```

# How EF Core Works Internally

```text
C# LINQ Query
      ↓
EF Core
      ↓
Converts to SQL
      ↓
SQL Server / PostgreSQL / MySQL
      ↓
Returns Data
      ↓
Converts to C# Objects
```

Example:

```csharp
var products = await _context.Products
    .Where(p => p.Price > 100)
    .ToListAsync();
```

Generated SQL:

```sql
SELECT * FROM Products
WHERE Price > 100
```

# Advantages of EF Core

## Faster Development

Less SQL writing.

## Database Independence

Same code works with:

- Microsoft SQL Server
- PostgreSQL
- MySQL
- SQLite

Only provider changes.

## Strongly Typed Queries

Compile-time checking.

```csharp
_context.Products.Where(x => x.Name == "Coffee")
```

Safer than raw SQL strings.

## Change Tracking

EF Core tracks modified entities automatically.

```csharp
product.Price = 500;

await _context.SaveChangesAsync();
```

EF Core automatically generates:

```sql
UPDATE Products
SET Price = 500
WHERE Id = 1
```

# EF Core vs ADO.NET

| Feature | EF Core | ADO.NET |
|---|---|---|
| Level | High-Level ORM | Low-Level Data Access |
| SQL Writing | Minimal | Manual |
| Development Speed | Fast | Slower |
| Performance | Slightly Lower | Faster |
| Control | Medium | Full |
| Best For | Business Apps | High-performance systems |

# EF Core Approaches

## 1. Code First

Most common.

You create C# classes first.

```text
C# Classes → Database
```

Uses migrations.

### Best for

- New projects
- Modern development

---

## 2. Database First

Existing database already exists.

```text
Database → Generate Models
```

# Important EF Core Concepts

## LINQ Queries

```csharp
var products = _context.Products
    .Where(x => x.Price > 100)
    .OrderBy(x => x.Name);
```

---

## Migrations

Used for database versioning.

```powershell
Add-Migration InitialCreate
Update-Database
```

---

## Relationships

EF Core handles:

- One-to-One
- One-to-Many
- Many-to-Many

Example:

```text
Category → Products
```

---

## Lazy Loading / Eager Loading

Controls related data loading.

```csharp
.Include(x => x.Category)
```

# Simple CRUD Example

## Insert

```csharp
var product = new Product
{
    Name = "Burger",
    Price = 300
};

_context.Products.Add(product);

await _context.SaveChangesAsync();
```

---

## Read

```csharp
var products = await _context.Products.ToListAsync();
```

---

## Update

```csharp
var product = await _context.Products.FindAsync(1);

product.Price = 400;

await _context.SaveChangesAsync();
```

---

## Delete

```csharp
var product = await _context.Products.FindAsync(1);

_context.Products.Remove(product);

await _context.SaveChangesAsync();
```
